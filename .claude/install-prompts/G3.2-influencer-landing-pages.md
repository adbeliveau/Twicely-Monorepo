## SCOPE -- EXACTLY WHAT TO BUILD

### 1. Database

**No new tables.** No schema changes. All data comes from existing tables:
- `affiliate` -- tier (must be INFLUENCER), status (must be ACTIVE), referralCode, applicationNote (contains serialized social links + audience size from G3.1)
- `user` -- name, displayName, bio, avatarUrl, username (joined via affiliate.userId)
- `promoCode` -- active promo codes for this affiliate (joined via promoCode.affiliateId, filtered by isActive=true and tier-appropriate)

### 2. Query: `getInfluencerLandingData`

**File:** `src/lib/queries/affiliate-landing.ts`

**Purpose:** Fetch all data needed to render an influencer landing page by slug.

**Logic:**
1. The slug in `/p/{slug}` is the influencer's `referralCode` (uppercased). Example: `/p/resalequeen` resolves to affiliate with `referralCode = 'RESALEQUEEN'`.
2. Query: find `affiliate` where `referralCode = slug.toUpperCase()` AND `tier = 'INFLUENCER'` AND `status = 'ACTIVE'`.
3. If not found, return null (page will show 404).
4. Join `user` table on `affiliate.userId` to get name, displayName, bio, avatarUrl.
5. Query active promo codes for this affiliate: `promoCode` where `affiliateId = affiliate.id` AND `isActive = true` AND (`expiresAt IS NULL OR expiresAt > now()`).
6. Parse `applicationNote` (JSON string containing `{ note, socialLinks, audienceSize }`) to extract social links for display.

**Return type:**
```typescript
interface InfluencerLandingData {
  affiliateId: string;
  referralCode: string;
  displayName: string | null;
  username: string | null;
  bio: string | null;
  avatarUrl: string | null;
  socialLinks: {
    instagram?: string;
    youtube?: string;
    tiktok?: string;
    blog?: string;
  } | null;
  promoCodes: Array<{
    code: string;
    discountType: string; // 'PERCENTAGE' | 'FIXED'
    discountValue: number; // BPS for percentage, cents for fixed
    durationMonths: number;
  }>;
}
```

### 3. Route: `/p/[slug]` (Dynamic Catch-All Within Policy Layout)

**CRITICAL ROUTING CONCERN:** The `/p/` prefix is ALREADY used for static policy pages (`/p/privacy`, `/p/terms`, `/p/fees`, `/p/buyer-protection`, `/p/how-it-works`, `/p/policies`). These are hardcoded pages in `src/app/(marketplace)/p/`.

**Resolution:** Add a dynamic `[slug]` route at `src/app/(marketplace)/p/[slug]/page.tsx`. Next.js App Router will prefer the static routes (`/p/privacy`, `/p/terms`, etc.) over the dynamic `[slug]` route. The dynamic route only catches slugs that DON'T match an existing static route.

**Important:** The existing static policy pages (`privacy`, `terms`, `fees`, `buyer-protection`, `how-it-works`, `policies`) must continue working unchanged. The dynamic `[slug]` route handles everything ELSE under `/p/`.

**NOT SPECIFIED -- owner decision needed:** What happens when someone hits `/p/nonexistent-slug` (not a policy page, not an influencer)? The recommended behavior is to return a 404 via `notFound()` from `next/navigation`. This is consistent with how `/i/[slug]` and `/st/[slug]` handle missing resources.

### 4. Page: `src/app/(marketplace)/p/[slug]/page.tsx`

**Gate:** PUBLIC (no auth required -- this is a marketing/landing page)

**Layout:** Uses existing policy layout (`src/app/(marketplace)/p/layout.tsx`) which provides `max-w-3xl px-4 py-12` centering. This is appropriate for a landing page.

**SSR Logic:**
1. Extract `slug` from params.
2. Call `getInfluencerLandingData(slug)`.
3. If null, call `notFound()` to render 404.
4. Render the influencer landing page component.

**SEO Metadata:**
- Title: `{displayName || username} on Twicely | Join with {referralCode}`
- Description: First 160 chars of bio, or fallback: `Join Twicely through {displayName || username}. Buy and sell secondhand fashion, electronics, and more.`
- noindex: false (these pages SHOULD be indexable -- they're marketing pages for influencer reach)
- OG image: avatarUrl if available

**Referral Attribution Flow:**
- The page renders a primary CTA button: "Join Twicely" that links to `/ref/{referralCode}` (the existing referral handler from G1.6).
- This ensures the existing first-touch attribution cookie logic is used. No need to duplicate cookie-setting logic.
- If the influencer has active promo codes, show them with a "Use code {CODE} for X% off" message. The user enters the code during subscription checkout (promo code redemption already works from G1.5).

### 5. Component: `src/components/affiliate/influencer-landing.tsx`

**Props:**

**Sections (top to bottom):**

**Styling:** Use existing shadcn/ui components (Card, Button, Avatar, Badge). No custom CSS. Tailwind classes only. The policy layout provides the outer container.

### Positive Cases
1. Visiting `/p/RESALEQUEEN` (where RESALEQUEEN is the referralCode of an ACTIVE INFLUENCER affiliate) renders a landing page with the influencer's display name, bio, avatar, social links, and active promo codes.
2. The "Join Twicely" CTA links to `/ref/RESALEQUEEN` which triggers the existing referral handler (cookie + redirect to signup).
3. Promo codes display the correct discount format: "20% off first 3 months" for PERCENTAGE type, "$10 off first month" for FIXED type.
4. Social links parsed from `applicationNote` JSON are displayed as clickable icon links.
5. Page generates correct SEO metadata (title, description, OG image).
6. Case-insensitive slug matching: `/p/resalequeen` and `/p/ResaleQueen` both resolve to the same page (slug is uppercased before lookup).
7. Existing policy pages (`/p/privacy`, `/p/terms`, `/p/fees`, `/p/buyer-protection`, `/p/how-it-works`, `/p/policies`) continue working unchanged.

11. Visiting `/p/PENDINGINFLUENCER` (where the affiliate has status=PENDING, not yet approved) returns 404.
12. When `affiliate.enabled` platform setting is false, ALL landing pages return 404.

14. Expired promo codes (expiresAt < now) are NOT displayed on the landing page.
15. Inactive promo codes (isActive = false) are NOT displayed on the landing page.

### Data Integrity
16. No internal IDs (affiliateId, userId, promoCodeId) appear in the rendered HTML.
17. The referralCode is the only identifier exposed in URLs and page content.
18. Discount values display correctly: BPS values converted to percentage for display (e.g., 2000 BPS -> "20%"), cents values converted to dollars for display (e.g., 5000 cents -> "$50").

### Authorization
19. The page is PUBLIC -- no auth required, no session check, no CASL check on the page itself.
20. No server actions on this page (it is read-only, SSR only).

### Vocabulary
21. No banned terms appear in any file (UI text, code comments, test descriptions).
22. Discount on subscriptions is described as "off your first X months" -- NOT "off your subscription" or "balance credit".

## IMPLEMENTATION NOTES

### How the referral flow works end-to-end with landing pages:

```
1. Influencer shares: twicely.co/p/RESALEQUEEN (or with UTMs)
2. Visitor sees landing page: influencer profile, value prop, promo codes
3. Visitor clicks "Join Twicely" -> redirects to /ref/RESALEQUEEN(?utm_source=...)
4. /ref/{code} handler (G1.6):
   a. Looks up affiliate by referralCode
   b. Creates referral record (CLICKED status)
   c. Sets first-touch attribution cookie (twicely_ref)
   d. Redirects to /auth/signup
5. User signs up -> referral updated to SIGNED_UP
6. User subscribes with promo code -> promo code applied, commission starts after trial
```

### Social links parsing from applicationNote:

The `applyForInfluencer` action (affiliate-influencer.ts) serializes social links into `applicationNote` as JSON:
```json
{
  "note": "I'm a resale educator with 50K YouTube subscribers...",
  "socialLinks": {
    "instagram": "https://instagram.com/resalequeen",
    "youtube": "https://youtube.com/@resalequeen"
  },
  "audienceSize": 50000
}
```

The query must safely parse this JSON. If parsing fails (e.g., old format, corrupted data), social links should be null -- not a page error.

### Discount display formatting:

```typescript
// PERCENTAGE type: discountValue is in BPS
// 2000 BPS -> "20% off"
// FIXED type: discountValue is in cents
// 5000 cents -> "$50 off"

function formatDiscount(type: string, value: number, durationMonths: number): string {
  const discountStr = type === 'PERCENTAGE'
    ? `${value / 100}% off`
    : `$${(value / 100).toFixed(2)} off`;
  const durationStr = durationMonths === 1
    ? 'your first month'
    : `your first ${durationMonths} months`;
  return `${discountStr} ${durationStr}`;
}
```
