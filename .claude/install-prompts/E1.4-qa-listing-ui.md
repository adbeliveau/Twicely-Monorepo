## Scope — Exactly What to Build

### Overview

This step builds **only UI components** and wires them into the existing listing detail page. No new database tables, no new server actions, no new queries, no new CASL rules. Everything backend already exists from E1.3.

### What to Build

#### 1. `QaSection` — Server Component (wrapper)

**Location:** `src/components/qa/qa-section.tsx`

Server component that:
1. Calls `getQuestionsForListing(listingId)` to fetch questions
2. Passes the data to the client component `QaPanel`
3. Also passes `listingId`, `currentUserId`, and `isOwnListing` (for conditional UI)

This is a thin server component whose only purpose is to do the async data fetch that client components cannot do.

Props:
```typescript
interface QaSectionProps {
  listingId: string;
  currentUserId: string | null;
  isOwnListing: boolean;
}
```

Behavior:
- Fetches questions via `getQuestionsForListing(listingId)`
- If zero questions AND the user is the listing owner, render nothing (no empty Q&A section for your own listing with no questions yet)
- Otherwise render `<QaPanel>` with the fetched data

#### 2. `QaPanel` — Client Component (interactive shell)

**Location:** `src/components/qa/qa-panel.tsx`

Client component that renders the entire Q&A section. Receives pre-fetched questions from the server component and manages local state for optimistic updates.

Props:
```typescript
interface QaPanelProps {
  listingId: string;
  currentUserId: string | null;
  isOwnListing: boolean;
  initialQuestions: QuestionSummary[];
}
```

Layout:
```
┌──────────────────────────────────────────┐
│  Questions & Answers ({count})           │
│                                          │
│  [AskQuestionForm] (if logged in,        │
│                     not own listing)      │
│                                          │
│  ┌─ Pinned ────────────────────────────┐ │
│  │  📌 Q: Does this come with charger? │ │
│  │      A: Yes, original 96W charger   │ │
│  │      [Pin] [Hide] (if owner)        │ │
│  └─────────────────────────────────────┘ │
│                                          │
│  ┌─ Question ──────────────────────────┐ │
│  │  Q: How many battery cycles?        │ │
│  │     Awaiting answer...              │ │
│  │     [Answer] [Pin] [Hide] (if owner)│ │
│  └─────────────────────────────────────┘ │
│                                          │
│  [Sign in to ask a question] (if guest)  │
└──────────────────────────────────────────┘
```

Behavior:
- Section header: "Questions & Answers" with count in parentheses, e.g., "Questions & Answers (5)"
- If user is logged in AND not the listing owner: show `AskQuestionForm` at the top
- If user is guest: show a link "Sign in to ask a question" pointing to `/auth/login`
- If user IS the listing owner: do NOT show ask form (sellers don't ask themselves questions)
- Questions render in order: pinned first (with pin indicator), then by newest first (this ordering is already done by `getQuestionsForListing`)
- Each question rendered via `QaQuestionCard`
- Empty state (no questions, not owner): "No questions yet. Be the first to ask!"
- Empty state (no questions, owner): component not rendered (handled in QaSection)

#### 3. `AskQuestionForm` — Client Component

**Location:** `src/components/qa/ask-question-form.tsx`

Props:
```typescript
interface AskQuestionFormProps {
  listingId: string;
}
```

Behavior:
- Textarea with placeholder "Ask the seller a question..."
- Character counter showing `{current}/500`
- Submit button "Ask Question" — disabled when textarea is empty or over 500 chars or while submitting
- On submit: call `askQuestion({ listingId, questionText })` server action

- On error: show error message below the form (e.g., "You already have 3 unanswered questions on this listing")
- Loading state: button shows "Submitting..." and is disabled

## Implementation Details

### qa-helpers.ts

```typescript
// src/components/qa/qa-helpers.ts

/**
 * Determine whether to show the "Ask a question" form.
 * Show if: user is logged in AND is NOT the listing owner.
 */
export function shouldShowAskForm(
  currentUserId: string | null,
  isOwnListing: boolean
): boolean {
  return currentUserId !== null && !isOwnListing;
}

/**
 * Determine whether to show seller action buttons (Answer, Pin, Hide).
 */
export function shouldShowSellerActions(isOwnListing: boolean): boolean {
  return isOwnListing;
}

/**
 * Format the section header label with question count.
 */
export function getQuestionCountLabel(count: number): string {
  return `Questions & Answers (${count})`;
}

/**
 * Determine whether the submit button should be disabled.
 */
export function isSubmitDisabled(
  text: string,
  maxLength: number,
  isSubmitting: boolean
): boolean {
  const trimmed = text.trim();
  return trimmed.length === 0 || trimmed.length > maxLength || isSubmitting;
}
```

### QaSection Server Component Pattern

```typescript
// src/components/qa/qa-section.tsx
import { getQuestionsForListing } from '@/lib/queries/qa';
import { QaPanel } from './qa-panel';

interface QaSectionProps {
  listingId: string;
  currentUserId: string | null;
  isOwnListing: boolean;
}

export async function QaSection({ listingId, currentUserId, isOwnListing }: QaSectionProps) {
  const questions = await getQuestionsForListing(listingId);

  // Don't show empty Q&A section on owner's own listing
  if (questions.length === 0 && isOwnListing) {
    return null;
  }

  return (
    <QaPanel
      listingId={listingId}
      currentUserId={currentUserId}
      isOwnListing={isOwnListing}
      initialQuestions={questions}
    />
  );
}
```

### UI Component Guidelines

#### AskQuestionForm
- Use `<Textarea>` from shadcn/ui (import from `@/components/ui/textarea`)
- Use `<Button>` from shadcn/ui (import from `@/components/ui/button`)
- Character counter: `<span className="text-sm text-muted-foreground">{text.length}/500</span>`
- When approaching limit (>450 chars), change counter color: `text-yellow-600`
- When over limit (>500 chars), change counter color: `text-destructive`
- Error display: `<p className="text-sm text-destructive">{error}</p>`
- Use `useRouter()` from `next/navigation` for `.refresh()` after successful submit
- Use `useTransition()` from React for pending state during server action call

#### QaQuestionCard
- Use a `<div>` with `border rounded-lg p-4` styling — consistent with other card patterns in codebase
- Pinned indicator: `<Pin className="h-4 w-4 text-primary" />` from lucide-react with label "Pinned"
- Asker name: `<span className="font-medium">{askerName}</span>`
- Relative time: `<span className="text-muted-foreground">{formatDate(createdAt, 'relative')}</span>`
- Unanswered state: `<p className="text-sm italic text-muted-foreground">Awaiting seller's answer</p>`
- Seller action buttons: use `<Button variant="ghost" size="sm">` for all three
- Hide confirmation: use `window.confirm()` for simplicity (no modal needed for a destructive-but-minor action). Text: "This will hide the question from all buyers. Continue?"
- Answer button icon: `<MessageSquare className="mr-1 h-4 w-4" />` from lucide-react
- Pin button icon: `<Pin className="mr-1 h-4 w-4" />`
- Hide button icon: `<EyeOff className="mr-1 h-4 w-4" />`

#### AnswerQuestionForm
- Same textarea/button pattern as AskQuestionForm but with 1000 char limit
- Placeholder: "Write your answer..."
- Submit button: "Submit Answer"
- Cancel button: `<Button variant="ghost" size="sm" onClick={onCancel}>Cancel</Button>`
