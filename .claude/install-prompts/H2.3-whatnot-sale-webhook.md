### 2.1 Whatnot Webhook Payload Types (`whatnot-types.ts` -- ADD to existing file)

Add these interfaces to the EXISTING `src/lib/crosslister/connectors/whatnot-types.ts`:

```typescript
/** Whatnot webhook envelope. All events share this structure. */
export interface WhatnotWebhookEnvelope {
  /** Unique event identifier (for idempotency/deduplication) */
  eventId: string;
  /** Event type, e.g. "order.completed" */
  eventType: string;
  /** ISO 8601 timestamp of when event was created on Whatnot */
  createdAt: string;
  /** Event-specific payload */
  data: Record<string, unknown>;
}

/** Whatnot order.completed webhook data payload */
export interface WhatnotOrderCompletedData {
  /** Whatnot order ID */
  orderId: string;
  /** The Whatnot listing ID that sold */
  listingId: string;
  /** Sale price as Money type */
  price: { amount: string; currencyCode: string };
  /** Buyer information */
  buyer: { id: string; username: string };
  /** ISO 8601 timestamp of when the sale completed */
  completedAt: string;
}
```

**Design note:** Whatnot's exact webhook payload structure is not publicly documented (Whatnot is a newer API partner). These interfaces are designed to match common patterns from their GraphQL API (Money type = `{ amount: string, currencyCode: string }`). The installer should structure the types so they can be easily adjusted once Whatnot confirms their exact webhook schema. The important thing is the handler logic -- the type shapes can be tuned later.

### 2.3 Webhook Signature Verification (`whatnot-webhook-verify.ts` -- NEW file)

Create `src/lib/crosslister/connectors/whatnot-webhook-verify.ts`:

This module handles HMAC-SHA256 signature verification for incoming Whatnot webhooks.

**Logic:**
1. Read `crosslister.whatnot.webhookSecret` from `platform_settings` table.
2. Compute HMAC-SHA256 of the raw request body using the webhook secret.
3. Compare the computed signature with the `X-Whatnot-Signature` header value using `timingSafeEqual`.
4. Return `{ valid: boolean; error?: string }`.

**Pattern:** Follow the Stripe webhook pattern (read raw body, verify signature before parsing JSON). Use Node.js `crypto.createHmac` and `crypto.timingSafeEqual` -- no external dependencies.

**Implementation details:**
- Function signature: `verifyWhatnotSignature(rawBody: string, signatureHeader: string): Promise<{ valid: boolean; error?: string }>`
- Loads the webhook secret from `platform_settings` (key: `crosslister.whatnot.webhookSecret`). If the secret is empty/missing, return `{ valid: false, error: 'Webhook secret not configured' }`.
- Compute: `crypto.createHmac('sha256', secret).update(rawBody, 'utf8').digest('hex')`
- Compare using `crypto.timingSafeEqual(Buffer.from(computed), Buffer.from(signatureHeader))` to prevent timing attacks.
- If lengths differ (timingSafeEqual requires same length), return `{ valid: false, error: 'Invalid signature' }`.

### 2.4 Whatnot Sale Webhook Handler (`sale-webhook-handler.ts` -- ADD to existing file)

Add `handleWhatnotSaleWebhook` to the EXISTING `src/lib/crosslister/handlers/sale-webhook-handler.ts` (currently 102 lines, has only eBay handler).

**Function signature:**
```typescript
export async function handleWhatnotSaleWebhook(
  envelope: WhatnotWebhookEnvelope
): Promise<void>
```

**Logic (mirrors `handleEbaySaleWebhook` pattern exactly):**

1. Parse `envelope.data` through `WhatnotOrderCompletedDataSchema.safeParse()`. If invalid, log warning and return.
2. Convert `data.price.amount` to integer cents using `parseMoneyToCents` (already exists in `whatnot-normalizer.ts`).
3. Validate: if `salePriceCents` is NaN or <= 0, log warning and return.
4. Look up `channelProjection` by `channel = 'WHATNOT'` AND `externalId = data.listingId`. If not found, log info ("not managed by Twicely") and return silently.
5. Compute platform fee: `await getPlatformFeeRate('WHATNOT')` then `calculatePlatformFee(salePriceCents, feeRateBps)`.
6. Build `DetectedSale` object with:
   - `listingId`: from the projection row
   - `projectionId`: from the projection row
   - `channel`: `'WHATNOT'`
   - `externalOrderId`: `data.orderId`
   - `salePriceCents`: converted cents value
   - `platformFeeCents`: computed above
   - `buyerUsername`: `data.buyer.username`
   - `soldAt`: `new Date(data.completedAt)`
7. Call `await handleDetectedSale(detectedSale)`.

**Imports needed (all already exist in the codebase):**
- `handleDetectedSale` from `../services/sale-detection`
- `getPlatformFeeRate, calculatePlatformFee` from `../services/platform-fees`
- `parseMoneyToCents` from `../connectors/whatnot-normalizer`
- `WhatnotOrderCompletedDataSchema` from `../connectors/whatnot-schemas`
- Types from `../connectors/whatnot-types`

### 2.5 API Route (`route.ts` -- NEW file)

Create `src/app/api/crosslister/whatnot/webhook/route.ts`:

**Endpoint:** `POST /api/crosslister/whatnot/webhook`

**Logic:**
1. Read raw body via `request.text()` (need raw body for signature verification).
2. Get `X-Whatnot-Signature` header from request. If missing, return 400.
3. Call `verifyWhatnotSignature(rawBody, signatureHeader)`. If invalid, return 400 `{ error: 'Invalid signature' }`.
4. Parse raw body as JSON. Validate through `WhatnotWebhookEnvelopeSchema.safeParse()`. If invalid, return 400.
5. Switch on `envelope.eventType`:
   - `'order.completed'`: Call `handleWhatnotSaleWebhook(envelope)`.
   - Any other event type: Log info ("unhandled event type") and return 200 (acknowledge but don't process).
6. Return 200 `{ received: true }`.
7. If `handleWhatnotSaleWebhook` throws, catch and log the error, but STILL return 200 (Lister Canonical Section 24.5: acknowledge receipt to prevent retries from overwhelming the system; we log the error for debugging).

**Error responses:**
- Missing signature header: 400
- Invalid signature: 400
- Invalid JSON / failed Zod validation: 400
- Handler error: 200 (acknowledge to prevent retry storms, log the error)

**No auth middleware required** -- this is a webhook endpoint called by Whatnot's servers, not by a Twicely user. Authentication is via HMAC signature verification only.

## 3. CONSTRAINTS -- WHAT NOT TO DO

### Banned Patterns
- NO `as any` or `@ts-ignore` -- use proper typing throughout
- NO hardcoded fee rates -- always read from `platform_settings` via `getPlatformFeeRate`
- NO money as floats -- `parseMoneyToCents` returns integer cents
- NO spreading request body into DB updates -- explicit field mapping only
- NO `console.log` -- use `logger` from `@/lib/logger`
- NO files over 300 lines (check `sale-webhook-handler.ts` after adding the Whatnot handler; if it exceeds 300 lines, the file must remain as-is per user preference to not split files)

### Banned Vocabulary
- NO `FVF`, `Final Value Fee` -- this step has no Twicely fee to name, but avoid even in comments
- NO `SellerTier`, `SubscriptionTier` -- not relevant to this step
- NO `wallet`, `balance`, `withdraw` in any seller-facing text

### Business Logic Constraints (CRITICAL)
- **NO Twicely fees on off-platform sales.** Decision #31, Pricing Canonical Section 1, Finance Engine Section 15. The `platformFeeCents` in the `DetectedSale` is the EXTERNAL platform's fee (Whatnot's cut), recorded purely for informational P&L tracking. Twicely charges $0 on this sale.
- **Off-platform sales do NOT go through Twicely checkout/escrow.** The webhook only updates crosslister state (listing status) and records informational ledger entries.
- **Emergency delists MUST fire.** When a Whatnot sale is detected, all other ACTIVE projections (eBay, Poshmark, etc.) must be delisted immediately. This is handled by `handleDetectedSale` which is already implemented -- the handler just needs to call it correctly.

### Gotchas
- **Whatnot Money type** is `{ amount: "12.99", currencyCode: "USD" }` -- a decimal STRING, not an integer. Must use `parseMoneyToCents` to convert (already exists in `whatnot-normalizer.ts`).
- **Webhook idempotency** is handled by `handleDetectedSale` (checks if projection is already SOLD). BUT the webhook handler should ALSO check the `eventId` for deduplication (Lister Canonical Section 24.5: "Webhook deduplication uses platform-provided event ID + timestamp"). Use the existing `channelProjection.status` check as the primary idempotency mechanism (same as eBay handler pattern), supplemented by logging the eventId for traceability.
- **Raw body for signature verification.** Must call `request.text()` BEFORE `request.json()`. The Stripe webhook routes follow this exact pattern (see `src/app/api/webhooks/stripe/route.ts` line 24).
- **Return 200 even on handler errors.** Webhook endpoints should acknowledge receipt to prevent the sending platform from retrying. Log errors internally but don't return 500.

## 4. ACCEPTANCE CRITERIA

### Positive Cases
1. A valid Whatnot `order.completed` webhook with correct HMAC signature is received, the listing exists in channelProjection as ACTIVE, and the handler: marks the canonical listing as SOLD, marks the Whatnot projection as SOLD, enqueues emergency delist jobs for all other ACTIVE projections, creates CROSSLISTER_SALE_REVENUE and CROSSLISTER_PLATFORM_FEE ledger entries, sends seller notification, and publishes Centrifugo event.
2. A valid webhook for a listing that Twicely does not manage (externalId not found in channelProjection) is silently acknowledged (200 response, no error logged).
3. A valid webhook with an already-SOLD projection is idempotently acknowledged (200 response, no duplicate processing).
4. An unhandled event type (e.g., `listing.updated`) returns 200 with `{ received: true }`.
5. The `crosslister.fees.whatnot.rateBps` platform setting is used for fee calculation when available; falls back to 1000 (10%) when not seeded.
6. `parseMoneyToCents("49.99")` correctly produces `4999` integer cents.
7. The webhook secret is read from `crosslister.whatnot.webhookSecret` in platform_settings.

### Negative Cases
8. A webhook with missing `X-Whatnot-Signature` header returns 400.
9. A webhook with an invalid HMAC signature returns 400.
10. A webhook with malformed JSON body returns 400.
11. A webhook with valid JSON but failing Zod envelope validation (e.g., missing `eventId`) returns 400.
12. A webhook with valid envelope but invalid `order.completed` data (e.g., missing `price.amount`) is logged as a warning and returns 200 (acknowledged but not processed).
13. A webhook with `price.amount = "0"` or `price.amount = "abc"` (invalid/zero price) is logged as a warning and returns 200.
14. **NO Twicely Transaction Fee is charged.** The only fees recorded are Whatnot's own platform fees, as informational ledger entries.
15. **NO escrow is created.** This sale happened on Whatnot, not on Twicely.

### Data Integrity
16. All monetary values stored as integer cents (salePriceCents, platformFeeCents).
17. Ledger entries created by `postOffPlatformSale` have `type = 'CROSSLISTER_SALE_REVENUE'` and `type = 'CROSSLISTER_PLATFORM_FEE'` with `channel = 'WHATNOT'`.
18. `WhatnotConnector.capabilities.hasWebhooks` is updated to `true`.

### Authorization
19. The webhook route has NO session/CASL auth check (it's authenticated via HMAC signature only -- this is correct for a platform-to-platform webhook).
