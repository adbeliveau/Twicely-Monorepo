### 2.4 Server Actions

Create `src/lib/actions/messaging.ts` (`'use server'`):

#### `createConversation(input: CreateConversationInput)`
Business rules (Feature Lock-in Section 19, Decision #38):
1. Authenticate via `authorize()`. Reject if no session.
2. CASL check: `ability.can('create', 'Conversation')`.
3. Validate input with `createConversationSchema.safeParse()`.
4. **Listing validation**: fetch listing by `listingId`. Verify `listing.status === 'ACTIVE'`. Reject if not found or not active.
5. **Cannot message own listing**: if `listing.ownerUserId === session.userId`, reject with "You cannot message yourself."
6. **Block check**: call `isBuyerBlocked(listing.ownerUserId, session.userId)`. If blocked, reject with "Unable to send message."
7. **One conversation per buyer-seller-listing** (Decision #38): query for existing conversation where `buyerId === session.userId AND sellerId === listing.ownerUserId AND listingId === input.listingId`. If found AND status is `OPEN`, return that conversation ID (reopen existing thread). If found AND status is `READ_ONLY` or `ARCHIVED`, reject with "This conversation is no longer active."
8. If no existing conversation, insert new conversation with explicit field mapping:
   - `listingId`: from input
   - `buyerId`: `session.userId`
   - `sellerId`: `listing.ownerUserId`
   - `subject`: `listing.title` (auto-generated from listing)
   - `status`: `'OPEN'`
   - `lastMessageAt`: `new Date()`
   - `buyerUnreadCount`: 0
   - `sellerUnreadCount`: 1
9. Insert the first message with explicit field mapping:
   - `conversationId`: new conversation ID
   - `senderUserId`: `session.userId`
   - `body`: from input
   - `attachments`: `[]` (no attachments on create)
   - `isAutoGenerated`: false
10. Fire-and-forget notification: `notifyNewMessage(messageId).catch(() => {})`.
11. `revalidatePath('/m')`.
12. Return `{ success: true, conversationId }`.

#### `sendMessage(input: SendMessageInput)`
Business rules:
1. Authenticate. Reject if no session.
2. CASL check: `ability.can('create', 'Message')`.
3. Validate input with `sendMessageSchema.safeParse()`.
4. **Conversation exists**: fetch conversation by `conversationId`. Reject if not found.
5. **Participant check**: verify `session.userId === conversation.buyerId OR session.userId === conversation.sellerId`. For delegated staff: `(session.onBehalfOfSellerId ?? session.userId) === conversation.sellerId`. If not a participant, reject "Not found."
6. **Conversation status check**: if `conversation.status !== 'OPEN'`, reject "This conversation is closed."
7. **Block check**: determine the other party. If sender is buyer, other is sellerId. If sender is seller, other is buyerId. Check `isBuyerBlocked()` in both directions (either party blocking the other). If blocked, reject "Unable to send message."
8. **Rate limit check**: count messages by this user in the last hour. If >= the value from `platform_settings` key `comms.messaging.rateLimitPerHour` (default 30, Feature Lock-in says 30/hr), reject "You are sending messages too quickly. Please try again later."
   - **SPEC INCONSISTENCY**: Feature Lock-in Section 19 says max 30/hour. Platform Settings Canonical Section 12.5 sets default to 20. **Resolution: Use the platform_settings value.** The seed will set it to 20 per the Platform Settings doc. The Feature Lock-in's "30" is a general rule; the platform_settings table is the runtime authority.
9. **Off-platform transaction detection** (Feature Lock-in Section 19): scan `body` for patterns: phone numbers (`\d{3}[-.\s]?\d{3}[-.\s]?\d{4}`), email addresses (`\S+@\S+\.\S+`), and keywords (`venmo`, `cashapp`, `paypal`, `wire transfer`, `zelle`, `cash app`). If detected:
   - Still send the message (Feature Lock-in: "Message still sends but is flagged").
   - Set `conversation.isFlagged = true`, `conversation.flagReason = 'Off-platform transaction detected'`.
10. Insert message with explicit field mapping:
    - `conversationId`: from input
    - `senderUserId`: `session.userId`
    - `body`: from input
    - `attachments`: from input (already validated as URL array, max 4)
    - `isAutoGenerated`: false
11. Update conversation: `lastMessageAt = new Date()`, `updatedAt = new Date()`. Increment the OTHER party's unread count:
    - If sender is buyer: `sellerUnreadCount = sellerUnreadCount + 1`
    - If sender is seller (or delegated staff): `buyerUnreadCount = buyerUnreadCount + 1`
12. Fire-and-forget notification: `notifyNewMessage(messageId).catch(() => {})`.
13. `revalidatePath('/m')`.
14. Return `{ success: true, messageId }`.

#### `markAsRead(input: MarkAsReadInput)`
Business rules:
1. Authenticate. Reject if no session.
2. Validate input.
3. Fetch conversation. Reject if not found.
4. **Participant check**: same as sendMessage.
5. Determine which unread counter to reset:
   - If current user is buyer: set `buyerUnreadCount = 0`.
   - If current user is seller (or delegated staff for that seller): set `sellerUnreadCount = 0`.
6. Bulk-update all unread messages in the conversation where `senderUserId !== currentUserId` AND `isRead === false`: set `isRead = true`, `readAt = new Date()`.
7. Update conversation `updatedAt`.
8. `revalidatePath('/m')`.
9. Return `{ success: true }`.

#### `archiveConversation(input: ArchiveConversationInput)`
Business rules:
1. Authenticate. Reject if no session.
2. CASL check: `ability.can('update', 'Conversation', { buyerId/sellerId condition })`.
3. Validate input.
4. Fetch conversation. Reject if not found.
5. **Participant check**: must be buyer or seller (or delegated staff with `messages.view` scope) of this conversation.
6. Set conversation `status = 'ARCHIVED'`, `updatedAt = new Date()`.
7. `revalidatePath('/m')`.
8. Return `{ success: true }`.

**IMPORTANT:** Keep all helper functions (off-platform detection, rate limit check, participant check) as **unexported** functions within the file to avoid accidentally exposing them as server actions. The `'use server'` directive at the top of the file means every exported function becomes a server action.

### 2.6 Off-Platform Detection Service

Create `src/lib/services/message-safety.ts`:

```typescript
// Off-platform transaction detection patterns
// Feature Lock-in Section 19: phone numbers, email addresses, external payment mentions

const OFF_PLATFORM_KEYWORDS = [
  'venmo', 'cashapp', 'cash app', 'paypal', 'wire transfer',
  'zelle', 'western union', 'money order', 'bitcoin', 'btc', 'crypto',
] as const;

// Phone pattern: 10+ digit sequences with optional separators
const PHONE_PATTERN = /\b\d{3}[-.\s]?\d{3}[-.\s]?\d{4}\b/;

// Simple email pattern
const EMAIL_PATTERN = /\b[^\s@]+@[^\s@]+\.[^\s@]+\b/;

export interface SafetyCheckResult {
  isFlagged: boolean;
  flagReason: string | null;
}

export function checkMessageSafety(body: string): SafetyCheckResult {
  const lowerBody = body.toLowerCase();

  // Check keywords
  for (const keyword of OFF_PLATFORM_KEYWORDS) {
    if (lowerBody.includes(keyword)) {
      return { isFlagged: true, flagReason: 'Off-platform transaction detected' };
    }
  }

  // Check phone numbers
  if (PHONE_PATTERN.test(body)) {
    return { isFlagged: true, flagReason: 'Off-platform transaction detected' };
  }

  // Check email addresses
  if (EMAIL_PATTERN.test(body)) {
    return { isFlagged: true, flagReason: 'Off-platform transaction detected' };
  }

  return { isFlagged: false, flagReason: null };
}
```

```typescript
import { db } from '@/lib/db';
import { conversation, message as messageTable, user } from '@/lib/db/schema';
import { listing } from '@/lib/db/schema';
import { eq } from 'drizzle-orm';
import { notify } from './service';
import { logger } from '@/lib/logger';

const BASE_URL = process.env.NEXT_PUBLIC_APP_URL ?? 'https://twicely.co';

/**
 * Notify the recipient when a new message is sent.
 * Fire-and-forget: errors are logged but never thrown.
 */
export async function notifyNewMessage(messageId: string): Promise<void> {
  try {
    // Fetch message
    const [msg] = await db
      .select({
        id: messageTable.id,
        conversationId: messageTable.conversationId,
        senderUserId: messageTable.senderUserId,
        body: messageTable.body,
        isAutoGenerated: messageTable.isAutoGenerated,
      })
      .from(messageTable)
      .where(eq(messageTable.id, messageId))
      .limit(1);

    if (!msg || msg.isAutoGenerated) return;

    // Fetch conversation
    const [conv] = await db
      .select({
        buyerId: conversation.buyerId,
        sellerId: conversation.sellerId,
        listingId: conversation.listingId,
      })
      .from(conversation)
      .where(eq(conversation.id, msg.conversationId))
      .limit(1);

    if (!conv) return;

    // Determine recipient
    const recipientId = msg.senderUserId === conv.buyerId
      ? conv.sellerId
      : conv.buyerId;

    // Do not notify if sender is the recipient (should not happen but guard)
    if (recipientId === msg.senderUserId) return;

    // Fetch sender name
    const [sender] = await db
      .select({ name: user.name })
      .from(user)
      .where(eq(user.id, msg.senderUserId))
      .limit(1);

    // Fetch listing title (if conversation has a listing)
    let itemTitle = 'an item';
    if (conv.listingId) {
      const [listingRow] = await db
        .select({ title: listing.title })
        .from(listing)
        .where(eq(listing.id, conv.listingId))
        .limit(1);
      if (listingRow?.title) {
        itemTitle = listingRow.title;
      }
    }

    await notify(recipientId, 'messaging.new_message', {
      senderName: sender?.name ?? 'Someone',
      itemTitle,
      messagePreview: msg.body.slice(0, 200),
      conversationUrl: `${BASE_URL}/m/${msg.conversationId}`,
    });
  } catch (err) {
    logger.error('[notifyNewMessage] Error', { error: String(err) });
  }
}
```

- NO UI components (pages, forms, conversation list, message thread)

### Authentication & Authorization

- [ ] `markAsRead` rejects unauthenticated users

## 9. SPEC INCONSISTENCIES — FLAGGED FOR RECORD

### Rate Limit Value
- **Feature Lock-in Section 19**: "max 30 messages per hour per user"
- **Platform Settings Canonical Section 12.5**: `comms.messaging.rateLimitPerHour` default = 20
- **Actors Canonical Section 6**: "Buyer | Message sends | 30/hour"
- **Resolution**: Platform Settings is the runtime authority. Seed the default as 20 (per Platform Settings Canonical). The rate limit is read from `platform_settings` at runtime, so admins can adjust to 30 if desired. The Feature Lock-in and Actors Canonical values represent the upper bound, not the default.

### E2.2 Build Tracker Name
- **Build Tracker**: E2.2 = "Meetup coordination messages"
- **User instruction**: E2.2 = "Buyer-seller messaging backend"
- **Resolution**: User instruction takes precedence. The messaging backend is the correct scope for E2.2. Meetup coordination messages will be a separate sub-step if needed.

### Conversation Subject in Actors Canonical
- **Actors Canonical Subject table**: Lists `Message` but NOT `Conversation` as a CASL subject
- **Codebase**: Only `Message` exists in subjects.ts, no `Conversation`
- **Resolution**: Add `Conversation` as a CASL subject. The spec groups message-level and conversation-level operations under the single `Message` subject, but implementation requires separate subjects for correct authorization granularity. Conversations need separate read/update conditions (buyerId vs sellerId) from individual messages (participantId).
