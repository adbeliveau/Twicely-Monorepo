### 2. ShopifyConnector Class

**File:** `src/lib/crosslister/connectors/shopify-connector.ts`

Implements `PlatformConnector` interface. Key differences from eBay/Etsy:

**Shopify OAuth is per-store, not per-user.**
- Each seller connects their **own Shopify store** (e.g., `my-vintage-store.myshopify.com`).
- The OAuth install URL is `https://{shop}.myshopify.com/admin/oauth/authorize`.
- The token exchange URL is `https://{shop}.myshopify.com/admin/oauth/access_token`.
- After authorization, Shopify returns a **permanent access token** (no refresh token, no expiry). The access token is valid until the app is uninstalled.
- The `externalAccountId` = the Shopify shop domain (e.g., `my-vintage-store.myshopify.com`).
- The `externalUsername` = the Shopify shop name.

**Connector properties:**
```typescript
readonly channel: ExternalChannel = 'SHOPIFY';
readonly tier: ConnectorTier = 'A';
readonly version = '1.0.0';
readonly capabilities: ConnectorCapabilities = {
  canImport: true,
  canPublish: true,
  canUpdate: true,
  canDelist: true,
  hasWebhooks: true,           // Shopify has robust webhook support
  hasStructuredCategories: true, // Shopify product types
  canAutoRelist: false,
  canMakeOffers: false,
  canShare: false,
  maxImagesPerListing: 250,     // Shopify supports up to 250 images per product
  maxTitleLength: 255,
  maxDescriptionLength: 65535,  // HTML body, effectively unlimited
  supportedImageFormats: ['jpg', 'jpeg', 'png', 'gif', 'webp'],
};
```

**Config loader:** `loadShopifyConfig()` reads from platform_settings:
- `crosslister.shopify.clientId` — Shopify app API key
- `crosslister.shopify.clientSecret` — Shopify app API secret key
- `crosslister.shopify.redirectUri` — OAuth callback URL
- `crosslister.shopify.scopes` — requested OAuth scopes
- `crosslister.shopify.apiVersion` — Shopify Admin API version (e.g., `2024-01`)

**Methods to implement (H3.1 scope = OAuth only; import/crosslist methods are stubs for H3.2/H3.3):**

1. **`buildAuthUrl(state: string, shopDomain?: string): Promise<string>`**
   - NOTE: This differs from other connectors because Shopify requires the shop domain in the auth URL.
   - The `state` param encodes a JSON with `{ csrfToken, shopDomain }` (shop domain must be known before auth).
   - Returns: `https://{shopDomain}/admin/oauth/authorize?client_id={apiKey}&scope={scopes}&redirect_uri={redirectUri}&state={state}`
   - If shopDomain is not provided in state, throw an error.

2. **`authenticate(credentials: AuthInput): Promise<AuthResult>`**
   - Token exchange: POST `https://{shopDomain}/admin/oauth/access_token` with `{ client_id, client_secret, code }`.
   - Shopify returns `{ access_token, scope }` (no expires_in, no refresh_token — token is permanent).
   - After getting the token, fetch shop info via GET `https://{shopDomain}/admin/api/{apiVersion}/shop.json` to populate externalAccountId and externalUsername.
   - Set `tokenExpiresAt: null` (permanent token).
   - Set `refreshToken: null` (Shopify doesn't use refresh tokens).

3. **`refreshAuth(account): Promise<AuthResult>`**
   - Shopify tokens don't expire. Return success with existing token.
   - If the token has been revoked (app uninstalled), this will be detected by a 401 response in other methods.

4. **`revokeAuth(account): Promise<void>`**
   - Call Shopify's access revoke endpoint: DELETE `https://{shopDomain}/admin/api_permissions/current.json` with the access token.
   - If the endpoint fails (404/401), log and continue (best-effort).

5. **`fetchListings(account, cursor?): Promise<PaginatedListings>`** — Stub returning empty. H3.2 scope.

6. **`fetchSingleListing(account, externalId): Promise<ExternalListing>`** — Stub throwing 'Not implemented'. H3.2 scope.

7. **`createListing(account, listing): Promise<PublishResult>`** — Stub returning `{ success: false, externalId: null, externalUrl: null, error: 'Not implemented — H3.3', retryable: false }`.

8. **`updateListing(account, externalId, changes): Promise<UpdateResult>`** — Stub.

9. **`delistListing(account, externalId): Promise<DelistResult>`** — Stub.

10. **`verifyListing(account, externalId): Promise<VerificationResult>`** — Stub.

11. **`healthCheck(account): Promise<HealthResult>`**
    - GET `https://{shopDomain}/admin/api/{apiVersion}/shop.json` with Bearer token.
    - Return healthy if 200 OK.

**Self-registration:** File ends with `registerConnector(new ShopifyConnector());`

**Shop domain extraction:** The shop domain is stored in `crosslisterAccount.externalAccountId`. Every method that makes API calls to Shopify extracts the shop domain from `account.externalAccountId`.

### 7. OAuth Callback Route

**File:** `src/app/api/crosslister/shopify/callback/route.ts`

Follows the exact pattern of `src/app/api/crosslister/whatnot/callback/route.ts`:

```
GET /api/crosslister/shopify/callback?code={code}&shop={shop}&state={state}&hmac={hmac}
```

**Key differences from other connectors:**
- Shopify includes `shop` and `hmac` query params in the callback.
- **HMAC verification:** Before exchanging the code, verify the callback by computing HMAC-SHA256 of the query string (minus the `hmac` param) using the Shopify API secret. This prevents CSRF and forged callbacks.
- The `shop` param provides the shop domain (e.g., `my-store.myshopify.com`).
- After successful auth, the `crosslisterAccount.externalAccountId` is set to the shop's `myshopify_domain`.

**Flow:**
1. Extract `code`, `shop`, `state`, `hmac` from query params.
2. Verify HMAC signature using `crosslister.shopify.clientSecret` from platform_settings.
3. Verify session (seller must be logged in via Better Auth).
4. CASL check: `ability.can('create', sub('CrosslisterAccount', { sellerId }))`.
5. Call `ShopifyConnector.authenticate()` with the code and shop domain.
6. Upsert `crosslisterAccount` row (same pattern as Whatnot callback).
7. Redirect to `/my/selling/crosslist/connect?connected=shopify` on success.
8. Redirect to `/my/selling/crosslist/connect?error=auth_failed` on failure.

**HMAC verification function:** Extract into a helper `verifyShopifyHmac(query: URLSearchParams, secret: string): boolean` within the route file (NOT exported to avoid accidental server action).

import type { Metadata } from 'next';
import { staffAuthorize } from '@/lib/casl/staff-authorize';
import { AdminPageHeader } from '@/components/admin/admin-page-header';
import { ConnectorSettingsPage } from '@/components/admin/settings/connector-settings-page';

## CONSTRAINTS -- WHAT NOT TO DO

1. **Do NOT install any Shopify SDK.** Use raw `fetch()` for all API calls, matching the pattern of every other connector.
2. **Do NOT implement import or crosslist logic.** Methods `fetchListings`, `fetchSingleListing`, `createListing`, `updateListing`, `delistListing`, `verifyListing` must be stubs returning empty/error results. These are H3.2 and H3.3 scope.
3. **Do NOT create webhook handler routes.** The webhook route (`/api/crosslister/shopify/webhook`) is mentioned in the admin page config for documentation purposes but is NOT built in H3.1.
4. **Do NOT use `as any` or `@ts-ignore`.** Fix the types.
5. **Do NOT hardcode fee rates.** All Shopify fee rates are read from `platform_settings`.
6. **Do NOT spread request body into DB updates.** Use explicit field mapping in the callback route.
7. **Do NOT export helper functions from the callback route.ts** (they would become unintended server actions).
8. **Do NOT use `storeId` as ownership key.** Always `userId` (stored as `sellerId` in crosslisterAccount).
9. **Do NOT create files over 300 lines.** The ShopifyConnector will be under 300 lines because import/crosslist methods are stubs.
10. **Token handling:** Shopify access tokens are permanent (no expiry, no refresh token). Set `tokenExpiresAt: null` and `refreshToken: null`.
11. **BANNED TERMS CHECK:** Ensure zero occurrences of: `SellerTier`, `SubscriptionTier`, `FVF`, `Final Value Fee`, `BASIC` (as StoreTier), `ELITE`, `PLUS` (as ListerTier), `MAX` (as ListerTier), `PREMIUM`, `Twicely Balance`, `wallet` (seller UI), `Withdraw` (seller UI).

28. Network error during authenticate returns `{ success: false, error: 'Network error...' }`.

30. No `console.log` in production code. Use `logger` from `@/lib/logger`.

32. `externalAccountId` = the `myshopify_domain` from Shopify shop info.
