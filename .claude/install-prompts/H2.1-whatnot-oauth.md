**Important Context:**
- Whatnot is a live selling + BIN (Buy It Now) marketplace focused on collectibles, trading cards, sneakers, vintage, and fashion.
- Whatnot launched a Seller API in 2024. It uses OAuth 2.0 + GraphQL (NOT REST like the other connectors).
- The Whatnot Seller API requires approved API access from the Whatnot team. This implementation scaffolds the full OAuth flow so it works immediately when real credentials are obtained.
- H2.1 covers ONLY OAuth connection. H2.2 (BIN crosslist) and H2.3 (sale webhooks) are separate future steps.

### 2.2 WhatnotConnector Class (Tier B OAuth, GraphQL)

Create `src/lib/crosslister/connectors/whatnot-connector.ts` implementing `PlatformConnector`.

**Key architectural difference from other Tier B connectors:** Whatnot uses a GraphQL API, not REST. The `fetchListings`, `createListing`, `updateListing`, and `delistListing` methods all use GraphQL mutations/queries against `https://api.whatnot.com/seller-api/graphql`.

**Whatnot API Details (from official Whatnot Seller API documentation):**

| Property | Value |
|----------|-------|
| Auth URL | `https://api.whatnot.com/seller-api/rest/oauth/authorize` |
| Token URL | `https://api.whatnot.com/seller-api/rest/oauth/token` |
| GraphQL endpoint | `https://api.whatnot.com/seller-api/graphql` |
| Staging Auth URL | `https://api.stage.whatnot.com/seller-api/rest/oauth/authorize` |
| Staging Token URL | `https://api.stage.whatnot.com/seller-api/rest/oauth/token` |
| Staging GraphQL | `https://api.stage.whatnot.com/seller-api/graphql` |
| Access token prefix | `wn_access_tk_` (production), `wn_access_tk_test_` (staging) |
| Refresh token expiry | 1 year |
| Scopes needed | `read:inventory`, `write:inventory`, `read:orders` |

**Connector properties:**

**Config loader pattern:** Follow exact same pattern as `loadMercariConfig()`:
```typescript
interface WhatnotConfig {
  clientId: string;
  clientSecret: string;
  redirectUri: string;
  environment: string; // 'PRODUCTION' or 'STAGING'
}

async function loadWhatnotConfig(): Promise<WhatnotConfig> {
  const rows = await db
    .select({ key: platformSetting.key, value: platformSetting.value })
    .from(platformSetting)
    .where(eq(platformSetting.category, 'crosslister'));

  const settingsMap = new Map<string, unknown>(rows.map((r) => [r.key, r.value]));

  return {
    clientId: String(settingsMap.get('crosslister.whatnot.clientId') ?? ''),
    clientSecret: String(settingsMap.get('crosslister.whatnot.clientSecret') ?? ''),
    redirectUri: String(
      settingsMap.get('crosslister.whatnot.redirectUri') ??
        'https://twicely.co/api/crosslister/whatnot/callback',
    ),
    environment: String(settingsMap.get('crosslister.whatnot.environment') ?? 'PRODUCTION'),
  };
}
```

**Environment-aware base URL helper:**
```typescript
function getBaseUrl(environment: string): string {
  return environment === 'STAGING'
    ? 'https://api.stage.whatnot.com/seller-api'
    : 'https://api.whatnot.com/seller-api';
}
```

**Methods to implement:**

1. `buildAuthUrl(state: string): Promise<string>` -- Build OAuth authorize URL with `client_id`, `redirect_uri`, `response_type=code`, `scope=read:inventory write:inventory read:orders`, `state`.

2. `authenticate(credentials: AuthInput): Promise<AuthResult>` -- Exchange authorization code for tokens via POST to token URL with `grant_type=authorization_code`, `client_id`, `client_secret`, `code`, `redirect_uri`. Then fetch user profile via GraphQL `me` query to get `externalAccountId` and `externalUsername`.

3. `refreshAuth(account: CrosslisterAccount): Promise<AuthResult>` -- POST to token URL with `grant_type=refresh_token`, `client_id`, `client_secret`, `refresh_token`. Note: Whatnot invalidates the used refresh token and returns a new one -- the new refresh token MUST be stored.

4. `revokeAuth(_account: CrosslisterAccount): Promise<void>` -- Log only (no revoke endpoint known).

5. `fetchListings(account: CrosslisterAccount, cursor?: string): Promise<PaginatedListings>` -- GraphQL query `listings` with pagination. Map to `ExternalListing[]`. This is a stub for H2.1 -- return empty results with a logger.info indicating "Whatnot listing fetch not implemented until H2.2".

6. `fetchSingleListing(account: CrosslisterAccount, externalId: string): Promise<ExternalListing>` -- GraphQL query `listing(id: $id)`. Stub for H2.1.

7. `createListing` / `updateListing` / `delistListing` -- GraphQL mutations. Stub for H2.1 -- return `{ success: false, error: 'Not implemented until H2.2', retryable: false }`.

8. `verifyListing` -- Return `{ exists: false, status: 'UNKNOWN', ... }` (stub).

9. `healthCheck(account: CrosslisterAccount): Promise<HealthResult>` -- GraphQL `me` query to verify the token works and measure latency.

**GraphQL query for `me` (profile):**
```graphql
query { me { id username } }
```

**Self-registration at module load:**
```typescript

```

### 2.6 OAuth Callback Route

Create `src/app/api/crosslister/whatnot/callback/route.ts`.

Follow EXACT pattern from `src/app/api/crosslister/mercari/callback/route.ts`:

1. GET handler.
2. Extract `code` from query params.
3. Verify session via Better Auth (`auth.api.getSession`).
4. CASL authorization check: `ability.can('create', sub('CrosslisterAccount', { sellerId }))`.
5. Call `connector.authenticate({ method: 'OAUTH', code, redirectUri: '' })`.
6. Upsert `crosslisterAccount` row -- check for existing `(sellerId, 'WHATNOT')`, update if exists, insert if not.
7. Redirect to success URL: `/my/selling/crosslist/connect?connected=whatnot`.
8. On failure redirect to: `/my/selling/crosslist/connect?error=auth_failed`.

**Channel value:** `'WHATNOT'` (matches the new enum value).

## 3. CONSTRAINTS -- WHAT NOT TO DO

### Banned Terms
- No `SellerTier`, `SubscriptionTier`, `FVF`, `BASIC`, `ELITE`, `PLUS`, `MAX`, `PREMIUM`, `Twicely Balance`, `wallet`, `Withdraw` anywhere in the code.

### Tech Stack
- No Prisma, no NextAuth, no Redis (use Valkey references if needed), no tRPC.
- GraphQL queries are written as inline template strings sent via `fetch` -- do NOT add a GraphQL client library (no Apollo, no urql, no graphql-request). Plain `fetch` with JSON body is sufficient.

### Architecture Rules
- `whatnot-connector.ts` is a plain TypeScript module, NOT a `'use server'` file.
- Self-registers via `registerConnector(new WhatnotConnector())` at module load.
- Config loaded from `platform_settings` table via `loadWhatnotConfig()` -- NEVER hardcode credentials.
- The callback route uses Better Auth (`auth.api.getSession`), NOT NextAuth.
- CASL check in callback uses `defineAbilitiesFor()` + `sub()`, matching the Mercari pattern.
- Ownership via `sellerId` (which equals `userId`), never `storeId`.
- All files under 300 lines.

### H2.1 Scope Boundaries
- Do NOT implement full listing import/export (that is H2.2).
- Do NOT implement sale webhooks (that is H2.3).
- Do NOT implement live auction/streaming features (that is post-launch per Decision Rationale #7).
- The `fetchListings`, `createListing`, `updateListing`, `delistListing` methods should be stubs that return empty/failure results with informative messages.
- `buildAuthUrl`, `authenticate`, `refreshAuth`, and `healthCheck` should be fully implemented.

### GraphQL Specifics
- GraphQL requests go to `{baseUrl}/graphql` via POST with `Content-Type: application/json`.
- Body shape: `{ query: string, variables?: Record<string, unknown> }`.
- Response shape: `{ data: T | null, errors?: Array<{ message: string }> }`.
- Use the `WhatnotGraphQLResponse<T>` generic type for all GraphQL responses.
- Use explicit type assertions (`as WhatnotGraphQLResponse<{ me: WhatnotUserProfile }>`) after `response.json()`.
