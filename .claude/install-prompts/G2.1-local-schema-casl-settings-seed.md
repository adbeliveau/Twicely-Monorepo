## CONSTRAINTS -- WHAT NOT TO DO

- Do NOT modify the `local.ts` schema file unless a column is genuinely missing vs the Schema doc
- Do NOT modify any existing enum values
- Do NOT add tables not in the Schema doc
- Do NOT seed platform settings that already exist (they are already in v32-platform-settings.ts)
- CASL conditions use `buyerId` and `sellerId` (not `userId`) because the `localTransaction` table uses those column names for the two parties
- Do NOT add `LocalTransaction` or `SafeMeetupLocation` as delegation scopes -- they are accessed via `orders.view` / `orders.manage` (local transactions are sub-resources of orders)
