## 7. BUSINESS RULES REFERENCE (For Downstream Phases)

These business rules from the Feature Lock-in doc apply to the tables created in A2.2 but should NOT be implemented now. They are documented here so downstream install prompts can reference them:

### listing_question (E1.3, E1.4)
- Questions have a 500-character max (enforced in app, not DB)
- Answers have a 1000-character max (enforced in app, not DB)
- Seller can pin up to 3 questions per listing (`isPinned` field, app-enforced limit)
- Seller or staff delegate can answer (tracked via `answeredBy` FK to user)
- Questions can be hidden by moderation or seller (`isHidden` field)
- New question triggers notification to seller (E1.3 scope)
- Answer triggers notification to asker (E1.3 scope)
- Public Q&A section displayed on listing detail page (E1.4 scope)

### curated_collection / curated_collection_item (G3.10)
- Staff-curated collections for the Explore/Discovery page
- Collections have optional seasonal date ranges (startDate/endDate)
- Collections can be published/unpublished (isPublished)
- Collections are ordered by sortOrder within the published set
- Items within a collection are ordered by sortOrder
- A listing can only appear once per collection (unique constraint on collection_id + listing_id)
- Staff user tracked via `curatedBy` (collection creator) and `addedBy` (per-item adder)

### offer_bundle_item (C2.3 -- already complete)
- Junction table linking a BUNDLE-type offer to its constituent listings
- Buyer selects 2+ items from same seller, proposes a bundle price
- Seller accepts/declines/counters the bundle as a whole
- If accepted: creates a single order with all items
- Bundle offers count as 1 offer toward the per-seller limit
- This table is already used by the bundle negotiation feature (C2.3 DONE)

### live_session / live_session_product (PL.1 -- Post-Launch)
- Design-only -- NO runtime code until 6-12 months post-launch
- Schema exists for forward planning and data model validation
- Build when 10K+ MAB and seller demand exists
- Tracks: seller, title, status (SCHEDULED/LIVE/ENDED/CANCELLED), viewer counts, stream URL
- Products featured during a session are tracked via junction table
- A listing can only be featured once per session (unique constraint on session_id + listing_id)

### Listing Video Fields (G3.11)
- R2 upload, thumbnail extraction, listing detail display
- Video duration: 15-60 seconds per platform settings
- Mobile video recording in listing creation (G1.7 depends on G3.11)
- No runtime code in this step -- video upload and display is G3.11 scope
