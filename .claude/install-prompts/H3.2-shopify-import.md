export const ShopifyProductSchema = z.object({
  id: z.number(),
  title: z.string(),
  body_html: z.string().nullable(),
  vendor: z.string().optional().default(''),
  product_type: z.string().optional().default(''),
  status: z.enum(['active', 'archived', 'draft']),
  tags: z.string().optional().default(''),

Replace the 15-line stub with a full implementation. Follow the exact pattern from `whatnot-normalizer.ts`.

**Interface:**

```typescript
export interface ShopifyNormalizedData {
  externalId: string;
  title: string;
  description: string;
  priceCents: number;

  listedAt: Date | null;
  soldAt: null;

  shippingType: null;
  shippingPriceCents: null;
  weight: number | null;          // First variant weight in grams
  dimensions: null;

```

**Functions to implement (all exported for testing):**

1. **`parseShopifyPrice(priceStr: string): number`**
   - `"29.99"` -> `2999`
   - `""` / `NaN` -> `0`
   - Use `parseFloat` then `Math.round(parsed * 100)`.

2. **`mapShopifyStatus(status: string): 'ACTIVE' | 'SOLD' | 'ENDED' | 'DRAFT'`**
   - `'active'` -> `'ACTIVE'`
   - `'draft'` -> `'DRAFT'`
   - `'archived'` -> `'ENDED'`
   - anything else -> `'ENDED'`

3. **`stripHtml(html: string): string`**
   - Strip HTML tags using `html.replace(/<[^>]*>/g, '').trim()`.
   - Do NOT add an HTML parsing library.

4. **`convertWeightToGrams(weight: number | null | undefined, unit: string | null | undefined): number | null`**
   - null/undefined/0 weight -> `null`
   - `'lb'` -> `Math.round(weight * 453.592)`
   - `'oz'` -> `Math.round(weight * 28.3495)`
   - `'kg'` -> `Math.round(weight * 1000)`
   - `'g'` or null/undefined unit -> `Math.round(weight)`

5. **`normalizeShopifyProduct(product: ShopifyProductParsed, shopDomain?: string): ShopifyNormalizedData`**

   Field mapping:

   | Shopify Field | Twicely Field | Transformation |
   |--------------|---------------|----------------|
   | `id` (number) | `externalId` | `String(product.id)` |
   | `title` | `title` | `product.title.trim().slice(0, 255)` |
   | `body_html` | `description` | `stripHtml(product.body_html ?? '')` |
   | `variants[0].price` | `priceCents` | `parseShopifyPrice(...)`. If no variants, `0`. |
   | (not in product) | `currencyCode` | `'USD'` (shop currency from OAuth, not on product) |
   | sum of `variants[].inventory_quantity` | `quantity` | `Math.max(1, sum)` |
   | (not available) | `condition` | `null` |
   | `vendor` | `brand` | `product.vendor.trim() \|\| null` |
   | `images` sorted by `position` | `images` | `{ url: img.src, isPrimary: idx === 0, sortOrder: idx }` |
   | `tags`, `product_type`, variant `sku`/`barcode` | `itemSpecifics` | Build Record. See rules below. |
   | `handle` + `shopDomain` | `url` | `https://{shopDomain}/products/{handle}` or `''` |
   | `status` | `status` | `mapShopifyStatus(product.status)` |
   | `created_at` | `listedAt` | Parse ISO. Invalid -> `null`. |
   | (not available) | `soldAt` | `null` |
   | `product_type` | `category` | `product.product_type.trim() \|\| null` |
   | `variants[0].weight` + `weight_unit` | `weight` | `convertWeightToGrams(...)` |
   | (not available) | `shippingType`, `shippingPriceCents`, `dimensions` | `null` |

   **itemSpecifics rules:**
   - If `product.tags` is non-empty: `itemSpecifics.tags = product.tags`
   - If `product.product_type` is non-empty: `itemSpecifics.productType = product.product_type.trim()`
   - If first variant has truthy `sku`: `itemSpecifics.sku = variant.sku`
   - If first variant has truthy `barcode`: `itemSpecifics.barcode = variant.barcode`

6. **`toExternalListing(normalized: ShopifyNormalizedData): ExternalListing`**
   - Maps every field. Exact same pattern as `whatnot-normalizer.ts:toExternalListing`.

**`parseShopifyLinkHeader` implementation:**
```typescript
export function parseShopifyLinkHeader(linkHeader: string | null): string | null {
  if (!linkHeader) return null;
  const parts = linkHeader.split(',');
  for (const part of parts) {
    if (part.includes('rel="next"')) {
      const urlMatch = part.match(/<([^>]+)>/);
      if (urlMatch?.[1]) {
        try {
          const url = new URL(urlMatch[1]);
          return url.searchParams.get('page_info');
        } catch {
          return null;
        }
      }
    }
  }
  return null;
}
```

**`fetchShopifyProducts` implementation:**

1. Build URL:
   - First page (no cursor): `https://{shopDomain}/admin/api/{apiVersion}/products.json?status=active&limit=50`
   - Subsequent pages (with cursor): `https://{shopDomain}/admin/api/{apiVersion}/products.json?page_info={cursor}&limit=50`
   - **CRITICAL:** When `page_info` is present, ALL other filter params like `status=active` are IGNORED by Shopify because the cursor encodes the original query.
2. Fetch with `X-Shopify-Access-Token: {accessToken}` header.
3. Handle errors:
   - 401: log warning, return empty `PaginatedListings`.
   - Any non-OK: log error, return empty `PaginatedListings`.
   - Network error: catch, log error, return empty `PaginatedListings`.
4. Parse `{ products: ShopifyProduct[] }` from response body.
5. Validate each product with `ShopifyProductSchema.safeParse()`. Skip invalid (log warning).
6. Normalize each valid product via `normalizeShopifyProduct(parsed.data, shopDomain)`.
7. Filter to status `'ACTIVE'` only.
8. Convert via `toExternalListing()`.
9. Extract next cursor via `parseShopifyLinkHeader(response.headers.get('link'))`.
10. Return `{ listings, cursor: nextCursor, hasMore: nextCursor !== null, totalEstimate: null }`.

**`fetchSingleShopifyProduct` implementation:**

1. Build URL: `https://{shopDomain}/admin/api/{apiVersion}/products/{externalId}.json`
2. Fetch with auth header.
3. If `!response.ok`, throw `Error(...)`.
4. Parse `{ product: ShopifyProduct }` from body.
5. Validate with `ShopifyProductSchema.safeParse()`. If invalid, throw.
6. Normalize and return `toExternalListing(normalizeShopifyProduct(parsed.data, shopDomain))`.

### Gotchas

1. **Shopify cursor pagination ignores other params.** When `page_info` is used in the URL, ALL other query params (like `status=active`) are IGNORED by Shopify because the cursor encodes the original query. First page: `?status=active&limit=50`. Subsequent pages: `?page_info=CURSOR&limit=50` ONLY.

2. **Shopify prices live on variants, not the product.** The product object has no top-level price. Price comes from `product.variants[0].price` which is a decimal string like `"29.99"`.

3. **Shopify `body_html` contains raw HTML.** Must strip tags for plain-text `description`. Use regex `/<[^>]*>/g`.

4. **Shopify `tags` is a single comma-separated string**, NOT an array.

5. **Shopify product IDs are numbers.** Must convert to `String(id)` for `externalId`.

6. **The normalizer needs `shopDomain` for URLs.** The connector passes `account.externalAccountId`. The normalizer dispatch does NOT have it -- URLs will be empty from the dispatch path. This is fine.

7. **Multi-variant products:** Use first variant's price, sum all variant quantities. Minimum quantity = 1.

8. **Shopify `status` is lowercase** (`active`, `archived`, `draft`). The normalizer maps to uppercase.

9. **Weight units vary.** Shopify variants can have weight in `lb`, `oz`, `kg`, or `g`. Normalize to grams.

10. **`ShopifyProductParsed` vs `ShopifyProduct`**: The `shopify-types.ts` has `ShopifyProduct` (interface). The `shopify-schemas.ts` will export `ShopifyProductParsed = z.infer<typeof ShopifyProductSchema>`. They should be structurally compatible. Use `ShopifyProductParsed` as the normalizer input type since it comes from Zod parsing.

## 9. SPEC GAPS AND DECISIONS

### NOT SPECIFIED -- Decisions Made by This Prompt

1. **Multi-variant price strategy:** Use first variant's price as canonical. Most resale items have a single variant ("Default Title"). Multi-variant products are rare in resale context.

2. **Multi-variant quantity strategy:** Sum all variant quantities. Total inventory across all variants represents true available quantity. Minimum 1.

3. **Currency code:** Default `'USD'`. Shopify products.json does not include currency -- it comes from the shop object fetched during OAuth. A future enhancement could store shop currency in `sessionData`.

4. **Product URL:** `https://{shopDomain}/products/{handle}`. If the store has a custom domain, the `.myshopify.com` URL may redirect. Acceptable for import purposes.

5. **Normalizer dispatch gap fix:** WHATNOT case was missing from normalizer-dispatch.ts since H2.1/H2.2. The Whatnot connector handles normalization inside `fetchListings`, but the dispatch is needed for the import pipeline's transform stage. Fixed as part of H3.2.

6. **Shopify not in Lister Canonical:** Section 9.1 lists only 8 platforms. Shopify added via build tracker H3.1-H3.4 without canonical spec. Prompt based on Shopify Admin REST API docs + established connector patterns.
