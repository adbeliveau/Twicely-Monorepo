## NOTES FOR INSTALLER

1. This step is unusually small because the heavy lifting was done in A2.1. The schema, migration, seed data, CASL wiring, and field additions all already exist. The primary new work is the 24 platform settings and verification tests.

2. The seed data in `seed-affiliates.ts` does NOT seed platform settings -- it only seeds demo affiliate/referral/commission/payout data. The platform settings are a separate concern managed through the `v32-platform-settings*` pipeline.

3. When verifying the schema, pay attention to one deviation: the canonical (Section 4.2) defines `referralCodeIdx` as `unique().on(table.referralCode)` on the `affiliate` table's index block, but the actual implementation uses `.unique()` directly on the column definition instead. Both achieve the same SQL constraint. This is acceptable.

4. The `affiliatePayout.status` column uses `text('status')` rather than a dedicated enum. This matches the canonical exactly -- the canonical specifies it as `text` with comment `// PENDING | PROCESSING | COMPLETED | FAILED`. This is a deliberate choice (low cardinality, simple states), not a bug.

5. G1.3 (Community affiliate self-serve signup) depends on G1.2 being marked complete. Once this step passes verification, G1.3 can proceed.
