## 3. SCOPE — EXACTLY WHAT TO BUILD

### 3.1 New Server Action: `testConnectorConnection`

**File:** `src/lib/actions/admin-connector-settings.ts` (MODIFY — add new export)

Add a `testConnectorConnection` server action that validates a connector's credentials are configured. This mirrors `testStripeConnection` in `admin-integrations.ts`.

**Logic:**
```
1. staffAuthorize() — CASL check: ability.can('read', 'Setting')
2. Validate input: z.object({ connectorCode: z.string().min(1).max(50) }).strict()
3. Look up platform settings for crosslister.{connectorCode}.*
4. Determine auth type:
   - OAuth connectors (ebay, etsy, mercari, depop, grailed, fbMarketplace):
     check that crosslister.{code}.clientId AND crosslister.{code}.clientSecret are non-empty strings
   - Session connectors (poshmark, therealreal):
     check that crosslister.{code}.apiBase is a non-empty string
5. Also check that at least one of importEnabled/crosslistEnabled/automationEnabled is true
6. Return { success: boolean, message: string, checks: { hasCredentials, hasEnabledFeature } }
```

**Zod Schema:**
```typescript
const testConnectorConnectionSchema = z.object({
  connectorCode: z.string().min(1).max(50),
}).strict();
```

**Response Shape:**
```typescript
{ success: boolean; message: string; checks?: { hasCredentials: boolean; hasEnabledFeature: boolean } }
| { error: string }
```

## 10. SPEC INCONSISTENCIES (for owner awareness)

### INCONSISTENCY 1: CASL Subject Name
- Connector pages use `ability.can('read', 'Setting')` as the CASL subject
- Actors/Security Canonical lists BOTH `Setting` (Section 4) and `PlatformSetting` (Section 5.5)
- Stripe page uses `ability.can('read', 'ProviderAdapter')` which is a different subject entirely
- **Resolution:** Keep existing pattern (`Setting` for connector pages, `ProviderAdapter` for Stripe) since both work and Stripe page needs provider-level access for secrets.

### INCONSISTENCY 2: Test Connection Scope
- Build tracker says "test connection" for all connector pages
- There are no actual external API calls to test (connector SDK implementations are Phase H)
- **Resolution:** `testConnectorConnection` validates that credentials/config are seeded in platform_settings, matching the pattern of `testStripeConnection` which also only validates key existence.

- `/cfg/integrations` IS listed (row 836 in Feature Lock-in)
- The pages exist in the codebase and work correctly
