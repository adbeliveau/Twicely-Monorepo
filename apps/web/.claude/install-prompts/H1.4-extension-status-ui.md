### 2.2 New: Extension Status Banner

For Tier C platforms (Poshmark, TheRealReal), session-based auth depends on the browser extension being installed and connected. The crosslister dashboard must show:

1. **Extension installed & connected**: Green indicator, "Extension connected" text, last heartbeat time
2. **Extension not detected**: Amber banner at the top of the page: "Install the Twicely browser extension to connect Poshmark and The RealReal. These platforms require session-based authentication via the extension." with a link to the Chrome Web Store (placeholder URL).
3. **Extension connected but session expired for a Tier C channel**: Platform card shows "Session expired -- open [Platform] in your browser with the extension active to reconnect"

**How to detect extension installation from the server side:**
- The extension sends periodic heartbeats to `POST /api/extension/heartbeat` every 5 minutes
- The heartbeat response returns `connectedChannels: string[]`
- **Server-side detection**: Check if any `crosslisterAccount` with `authMethod = 'SESSION'` has `lastAuthAt` within the last 30 minutes (extension was recently active). This is a heuristic, not exact.
- **Client-side detection**: Use the H1.4-specific query parameter `?source=extension` that the extension sets when opening the crosslister page on install (see service-worker.ts line 10). This confirms extension presence for that page load.

**Important**: The extension itself is NOT a Next.js app. It is a standalone Chrome MV3 extension with plain TypeScript/HTML. The status UI on the server side can ONLY infer extension status from the data the extension sends to the API routes.

### 2.4 New Query: Extension Heartbeat Status

Create a lightweight query to check recent extension activity:

```typescript
// src/lib/queries/extension-status.ts
export async function getExtensionStatus(sellerId: string): Promise<{
  hasExtension: boolean;
  lastHeartbeatAt: Date | null;
  tierCAccounts: Array<{
    channel: string;
    status: string;
    lastAuthAt: Date | null;
    sessionExpired: boolean;
  }>;
}>
```

Logic:
- Query `crosslisterAccount` where `sellerId` matches AND `authMethod = 'SESSION'`
- `hasExtension` = true if ANY session-based account has `lastAuthAt` within last 30 minutes
- `sessionExpired` = `status === 'REAUTHENTICATION_REQUIRED'` OR `lastAuthAt` is null OR `lastAuthAt` is older than 24 hours (Tier C sessions are volatile)
- Return the list of Tier C accounts with their status

### 2.7 CASL Authorization

The page server component must check CASL before fetching data:

```typescript
import { authorize } from '@/lib/casl/authorize';

const session = await getSession();
if (!session?.user) redirect('/auth/login?callbackUrl=/my/selling/crosslist');

const ability = await authorize(session.user.id);
if (!ability.can('read', 'CrosslisterAccount')) {
  redirect('/my');
}
```

**Delegation**: Staff with `crosslister.read` scope can view this page (read-only). Staff with `crosslister.manage` scope can connect/disconnect platforms. The page itself requires only `read` -- action buttons within child components check `manage` permissions.

## 9. SPEC GAPS / OWNER DECISIONS NEEDED

### 9.1 Chrome Web Store Extension URL

NOT SPECIFIED -- The Chrome Web Store listing does not exist yet. Use a placeholder URL: `https://chromewebstore.google.com/detail/twicely-crosslister/PLACEHOLDER_ID`. Mark with a `TODO` comment for replacement when the extension is published.

### 9.2 Extension Heartbeat Threshold

The 30-minute threshold for `hasExtension` detection is a reasonable heuristic (heartbeat fires every 5 minutes per `HEARTBEAT_INTERVAL_MINUTES` in constants.ts). This is NOT specified in any canonical doc. Owner decision: is 30 minutes the right window, or should it be shorter (15 min) or longer (1 hour)?

### 9.3 Session Expiry Threshold

The 24-hour threshold for marking a Tier C session as "expired" is an implementation decision. The Lister Canonical Section 9.4 says "If session is invalidated (platform forces logout), status -> REAUTHENTICATION_REQUIRED" but does not specify a time-based expiry for sessions that simply go stale. The server session route sets `lastAuthAt` on each session update. Owner decision: is 24 hours appropriate for session staleness?

### 9.4 FB Marketplace Extension Status

FB Marketplace is Tier B (OAuth) in the channel registry, NOT Tier C. However, the extension DOES capture FB session data (informational, not auth-critical). The extension status banner should ONLY reference true Tier C platforms (Poshmark, TheRealReal) that REQUIRE the extension. FB Marketplace connects via OAuth and does not need the extension. The banner should NOT mention FB Marketplace.

### 9.5 Crosslister Dashboard vs. Crosslister Overview

Page Registry #57 says the crosslister page shows "Connected accounts, publish counts, job status." The Unified Hub Canonical sidebar lists this as the "Platforms" link (`/my/selling/crosslist/connect`). The crosslister dashboard at `/my/selling/crosslist` (route #57) is the overview, while `/my/selling/crosslist/connect` (route #58) is the connection management page. This install prompt builds ONLY route #57 (the dashboard/overview). Route #58 (connect page) should already exist or will be built separately.
