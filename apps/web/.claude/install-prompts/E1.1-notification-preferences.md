## 3. SPEC GAP — Schema Does Not Have Digest/Quiet-Hours Columns

**CRITICAL FINDING:** The canonical schema (`TWICELY_V3_SCHEMA_v2_0_7.md` Section 10.2) defines `notificationPreference` with ONLY these columns:

```
id, userId, templateKey, email, push, inApp, sms, createdAt, updatedAt
```

There are NO columns for:
- `digestFrequency` (daily/weekly)
- `digestTimeUtc` (user's preferred digest send time)
- `quietHoursStart` / `quietHoursEnd`
- `timezone`

But Feature Lock-in Section 27 REQUIRES these features. The digest/quiet-hours settings are user-level (not per-template), so they belong on a separate per-user settings row, NOT on the per-template `notificationPreference` table.

### Resolution: Add columns to `notificationPreference` table? Or create new table?

The per-template `notificationPreference` table is the wrong place for user-level settings (digest frequency applies to ALL informational notifications, quiet hours applies to ALL push/SMS). Two options:

**Option A (RECOMMENDED):** Add user-level notification settings columns to a new `notification_setting` row-per-user table. This is clean separation: `notificationPreference` = per-template channel overrides, `notificationSetting` = user-level global notification behavior.

**Option B:** Add columns to the `user` table. Simpler but pollutes the user table with notification concerns.

**NOT SPECIFIED — Owner decision needed:**
The schema doc has no table for user-level notification settings (digest, quiet hours, timezone). Feature Lock-in Section 27 requires them. The installer should use **Option A: create a `notification_setting` table** with one row per user, containing digest and quiet-hours configuration. This is a new table NOT in the schema doc — the owner must approve.

### Proposed `notification_setting` Table

```typescript
export const notificationSetting = pgTable('notification_setting', {
  id:                  text('id').primaryKey().$defaultFn(() => createId()),
  userId:              text('user_id').notNull().references(() => user.id, { onDelete: 'cascade' }).unique(),
  digestFrequency:     text('digest_frequency').notNull().default('daily'),    // 'daily' | 'weekly'
  digestTimeUtc:       text('digest_time_utc').notNull().default('14:00'),     // HH:MM in UTC (default = 9 AM ET)
  timezone:            text('timezone').notNull().default('America/New_York'), // IANA timezone string
  quietHoursEnabled:   boolean('quiet_hours_enabled').notNull().default(false),
  quietHoursStart:     text('quiet_hours_start'),                              // HH:MM in local time, e.g., '22:00'
  quietHoursEnd:       text('quiet_hours_end'),                                // HH:MM in local time, e.g., '08:00'
  dailySalesSummary:   boolean('daily_sales_summary').notNull().default(false), // Seller-only opt-in
  staleListingDays:    integer('stale_listing_days'),                           // null = disabled, e.g., 30
  trustScoreAlerts:    boolean('trust_score_alerts').notNull().default(false),  // Seller-only
  createdAt:           timestamp('created_at', { withTimezone: true }).notNull().defaultNow(),
  updatedAt:           timestamp('updated_at', { withTimezone: true }).notNull().defaultNow(),
});
```

**Migration needed:** One new migration file adding this table.

### 4.2 Zod Validation Schemas

Create `src/lib/validations/notification-settings.ts`:

```typescript
import { z } from 'zod';

const timeRegex = /^([01]\d|2[0-3]):([0-5]\d)$/; // HH:MM

export const digestSettingsSchema = z.object({
  digestFrequency: z.enum(['daily', 'weekly']),
  digestTimeUtc: z.string().regex(timeRegex, 'Must be HH:MM format'),
  timezone: z.string().min(1).max(50), // IANA timezone
}).strict();

export const quietHoursSchema = z.object({
  quietHoursEnabled: z.boolean(),
  quietHoursStart: z.string().regex(timeRegex, 'Must be HH:MM format').nullable(),
  quietHoursEnd: z.string().regex(timeRegex, 'Must be HH:MM format').nullable(),
}).strict().refine(
  (data) => {
    if (data.quietHoursEnabled) {
      return data.quietHoursStart !== null && data.quietHoursEnd !== null;
    }
    return true;
  },
  { message: 'Start and end times required when quiet hours enabled' }
);

export const sellerNotificationSettingsSchema = z.object({
  dailySalesSummary: z.boolean(),
  staleListingDays: z.number().int().min(1).max(365).nullable(),
  trustScoreAlerts: z.boolean(),
}).strict();

export const marketingOptInSchema = z.object({
  marketingOptIn: z.boolean(),
}).strict();

// Combined update schema for the full settings form
export const updateNotificationSettingsSchema = z.object({
  digestFrequency: z.enum(['daily', 'weekly']),
  digestTimeUtc: z.string().regex(timeRegex, 'Must be HH:MM format'),
  timezone: z.string().min(1).max(50),
  quietHoursEnabled: z.boolean(),
  quietHoursStart: z.string().regex(timeRegex).nullable(),
  quietHoursEnd: z.string().regex(timeRegex).nullable(),
  marketingOptIn: z.boolean(),
  // Seller-only fields (ignored if user is not seller)
  dailySalesSummary: z.boolean().optional(),
  staleListingDays: z.number().int().min(1).max(365).nullable().optional(),
  trustScoreAlerts: z.boolean().optional(),
}).strict().refine(
  (data) => {
    if (data.quietHoursEnabled) {
      return data.quietHoursStart !== null && data.quietHoursEnd !== null;
    }
    return true;
  },
  { message: 'Start and end times required when quiet hours enabled' }
);

export type UpdateNotificationSettingsInput = z.infer<typeof updateNotificationSettingsSchema>;
```

### 4.5 Enhance `notify()` Service

**Modify:** `src/lib/notifications/service.tsx`

The existing `notify()` function must be enhanced to respect:

1. **Quiet hours check**: Before sending PUSH or SMS, check if the user has quiet hours enabled and if the current time (in user's timezone) falls within their quiet hours window. If so, skip PUSH/SMS (in-app and email still proceed). **Exception**: if the template priority is `'CRITICAL'`, bypass quiet hours.

2. **Digest routing for Informational priority**: If a template has `priority: 'NORMAL'` or `priority: 'LOW'` (these are the "Informational" tier per Feature Lock-in), and the user has digest enabled, do NOT send the email immediately. Instead, mark the notification row with `channel: 'EMAIL'` but `sentAt: null` — it will be picked up by the digest cron job. For now, just add the routing logic; the actual digest email aggregation cron is out of scope for E1.1 (it requires BullMQ infrastructure that may not exist yet).

The logic flow becomes:

```
notify(userId, templateKey, data):
  template = TEMPLATES[templateKey]
  prefs = getUserPreferences(userId, templateKey)   // per-template channel toggles
  settings = getNotificationSettings(userId)         // user-level digest/quiet config

  For each enabled channel:
    IN_APP: always create immediately (no quiet hours affect in-app)
    EMAIL:
      if template.priority in ['NORMAL', 'LOW'] and settings.digestFrequency exists:
        create notification row with sentAt=null (queued for digest)
      else:
        send email immediately (as before)
    PUSH:
      if isQuietHours(settings) and template.priority !== 'CRITICAL':
        skip (do not create row — or create with failureReason='quiet_hours')
      else:
        send push (placeholder — push not implemented yet)
    SMS:
      same quiet hours logic as PUSH
```

**IMPORTANT:** Push and SMS sending are NOT implemented yet (only EMAIL and IN_APP exist). The quiet hours logic should be added now so it's ready when push/SMS are wired. For now, the quiet hours check returns early for PUSH/SMS since those channels have no send mechanism.

```typescript
/**
 * Check if the current time falls within the user's quiet hours.
 * Handles overnight ranges (e.g., 22:00-08:00).
 */
function isQuietHours(
  settings: { quietHoursEnabled: boolean; quietHoursStart: string | null; quietHoursEnd: string | null; timezone: string }
): boolean {
  if (!settings.quietHoursEnabled || !settings.quietHoursStart || !settings.quietHoursEnd) {
    return false;
  }

  // Get current time in user's timezone as HH:MM
  const now = new Date();
  const formatter = new Intl.DateTimeFormat('en-US', {
    hour: '2-digit',
    minute: '2-digit',
    hour12: false,
    timeZone: settings.timezone,
  });
  const parts = formatter.formatToParts(now);
  const hour = parts.find(p => p.type === 'hour')?.value ?? '00';
  const minute = parts.find(p => p.type === 'minute')?.value ?? '00';
  const currentMinutes = parseInt(hour) * 60 + parseInt(minute);

  const [startH, startM] = settings.quietHoursStart.split(':').map(Number);
  const [endH, endM] = settings.quietHoursEnd.split(':').map(Number);
  const startMinutes = startH * 60 + startM;
  const endMinutes = endH * 60 + endM;

  if (startMinutes <= endMinutes) {
    // Same day range (e.g., 14:00-16:00)
    return currentMinutes >= startMinutes && currentMinutes < endMinutes;
  } else {
    // Overnight range (e.g., 22:00-08:00)
    return currentMinutes >= startMinutes || currentMinutes < endMinutes;
  }
}
```

```typescript
export async function notify(
  userId: string,
  templateKey: TemplateKey,
  data: Record<string, string>
): Promise<void> {
  const template = TEMPLATES[templateKey];
  if (!template) return;

  const prefs = await getUserPreferences(userId, templateKey);
  const settings = await getNotificationSettingsInternal(userId); // lightweight internal version
  const subject = interpolate(template.subjectTemplate, data);
  const body = interpolate(template.bodyTemplate, data);

  // IN_APP — always immediate, never affected by quiet hours
  if (prefs.inApp) {
    await createInAppNotification(userId, template, templateKey, subject, body, data);
  }

  // EMAIL — digest routing for informational priority
  if (prefs.email) {
    const isInformational = template.priority === 'NORMAL' || template.priority === 'LOW';
    if (isInformational && settings.digestFrequency) {
      // Queue for digest (create row with sentAt=null)
      await createDigestQueuedNotification(userId, template, templateKey, subject, body, data);
    } else {
      // Send immediately
      await sendEmailNotification(userId, template, templateKey, subject, body, data);
    }
  }

  // PUSH — quiet hours check (push sending not implemented yet)
  if (prefs.push) {
    const isCritical = template.priority === 'CRITICAL';
    if (!isCritical && isQuietHours(settings)) {
      // Skip — user is in quiet hours
      return;
    }
    // TODO: implement push sending when infrastructure is ready
  }

  // SMS — quiet hours check (SMS sending not implemented yet)
  if (prefs.sms) {
    const isCritical = template.priority === 'CRITICAL';
    if (!isCritical && isQuietHours(settings)) {
      return;
    }
    // TODO: implement SMS sending when infrastructure is ready
  }
}
```
