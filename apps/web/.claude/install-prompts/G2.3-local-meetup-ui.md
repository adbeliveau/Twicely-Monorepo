## SCOPE -- EXACTLY WHAT TO BUILD

### 1. Seller Local Settings Page

**Route:** `/my/selling/settings/local` (NOTE: Page Registry does not explicitly list this route, but Local Canonical Section 3 says "UI at `/my/selling/settings/local`". Follow the Canonical.)

**File:** `src/app/(dashboard)/my/selling/settings/local/page.tsx` (NEW)

- SSR page, requires SELLER auth
- Displays current `maxMeetupDistanceMiles` from sellerProfile
- Form with:
  - Toggle: "Enable local pickup for my listings" (enables/disables by setting distance to non-null/null)
  - Slider or number input: "Maximum meetup distance" (1-50 miles)
  - Default radius display: reads from `commerce.local.defaultRadiusMiles` platform setting
  - Max radius: reads from `commerce.local.maxRadiusMiles` platform setting
- Submit calls `updateSellerLocalSettings` action from G2.2
- Shows info text: "When enabled, buyers within your radius can choose local pickup instead of shipping."

**File:** `src/components/local/seller-local-settings-form.tsx` (NEW)
- Client component (`'use client'`)
- shadcn/ui: Switch, Slider or Input (number), Button, Card
- Shows "Save" button, handles pending state
- Uses `useTransition` for server action call

### 2. Local Pickup Meetup Detail on Order Page (Buyer Side)

**File:** `src/components/local/local-meetup-card.tsx` (NEW)
- Client component for displaying local meetup information on the order detail page
- Shows when `order.isLocalPickup === true`
- Displays:
  - Meetup status badge (from localTransaction.status)
  - Scheduled date/time (from localTransaction.scheduledAt)
  - Meetup location name + address (if meetupLocationId set, from joined safeMeetupLocation)
  - QR code display area (for seller to show): uses the `confirmationCode` to generate a QR code image
  - Manual code display (for seller): shows the 6-digit `offlineCode`
  - Check-in button: calls `checkInToMeetupAction`
  - Confirm receipt button (buyer only): shows after scanning QR or entering manual code
  - Status timeline: SCHEDULED -> CHECKED_IN -> RECEIPT_CONFIRMED -> COMPLETED

**QR Code Generation:**
- Use a lightweight QR code library. Recommended: `qrcode` npm package (generates QR code as data URL)
- The QR code encodes the `confirmationCode` string (cuid2 value)
- Seller sees QR code on their phone; buyer scans it
- Buyer's scan triggers `confirmReceiptAction` with the decoded code

**File:** `src/components/local/local-meetup-timeline.tsx` (NEW)
- Displays the local transaction status as a vertical timeline
- States: Scheduled -> Seller Checked In -> Buyer Checked In -> Both Checked In -> Receipt Confirmed -> Completed
- Current state highlighted, past states checked

**File:** `src/components/local/qr-code-display.tsx` (NEW)
- Takes `code: string` prop
- Renders QR code image using `qrcode` library
- Includes "Can't scan? Use code: XXXXXX" text below with the 6-digit offline code
- Seller shows this screen at meetup

**File:** `src/components/local/manual-code-entry.tsx` (NEW)
- Buyer's fallback: 6-digit code input field
- shadcn/ui: Input (6 chars, numeric pattern), Button ("Confirm Receipt")
- On submit: calls `confirmReceiptAction` with method='MANUAL'

## CONSTRAINTS -- WHAT NOT TO DO

- Do NOT build the QR scanner (camera) component in this step -- the buyer will use their phone's native camera or a separate scan step. For now, the confirm action takes a code string input. A scan-to-confirm UX is G2.4.
- Do NOT implement messaging integration (quick-reply templates) -- that is G2.4
- Do NOT implement the safety timer -- that is G2.6
- Do NOT implement no-show detection -- that is G2.5
- Do NOT modify the checkout flow in this step -- the fulfillment type selection at checkout already exists from B3.4. This step focuses on the POST-checkout meetup flow.
- Keep all components under 300 lines
- Use shadcn/ui components only (Card, Badge, Button, Input, Switch, etc.)
- QR code library must be installed: `qrcode` (check if already in package.json first)
- Money display: use `formatCentsToDollars` from `src/lib/finance/format.ts`
- All date display: use relative time or formatted locale string, not raw ISO
