### 2.2 New Helper: `whatnot-graphql.ts`

Create a reusable GraphQL request helper to avoid duplicating fetch+parse logic across 6 methods. This also keeps the connector file from exploding past 300 lines (it is already at 362 lines with stubs).

```typescript
// src/lib/crosslister/connectors/whatnot-graphql.ts

/**
 * Low-level GraphQL helper for Whatnot Seller API.
 * Handles: environment-aware URL resolution, auth header injection,
 * error extraction, and response typing.
 * NOT a 'use server' file.
 */

interface WhatnotGraphQLRequestOptions {
  accessToken: string;
  environment: string; // 'PRODUCTION' | 'STAGING'
  query: string;
  variables?: Record<string, unknown>;
}

interface WhatnotGraphQLResult<T> {
  data: T | null;
  errors: Array<{ message: string; path?: string[] }> | null;
  status: number;
}
```

**Functions to export:**
- `executeGraphQL<T>(options: WhatnotGraphQLRequestOptions): Promise<WhatnotGraphQLResult<T>>` -- single fetch call with proper headers, JSON parse, error extraction. Returns `{ data, errors, status }`. Does NOT throw -- caller decides how to handle errors.
- `getGraphQLUrl(environment: string): string` -- returns `https://api.whatnot.com/seller-api/graphql` or staging variant.

**Rules:**
- Uses plain `fetch()` -- no GraphQL client library.
- Content-Type: `application/json`.
- Authorization: `Bearer ${accessToken}`.
- On HTTP errors (non-200), still tries to parse body for GraphQL error messages.
- Logs all errors via `logger.error()` with `[WhatnotGraphQL]` prefix.
- Never exposes raw error messages to callers beyond the GraphQL error text.

### 2.4 GraphQL Mutations (for createListing / updateListing / delistListing)

**Whatnot BIN listing lifecycle (2-step create):**
1. `listingCreate` -- creates a DRAFT listing
2. `listingPublish` -- transitions DRAFT to PUBLISHED (live on Whatnot)

For updates: `listingUpdate` (works on PUBLISHED listings too).
For delist: `listingUnpublish` (transitions PUBLISHED to UNPUBLISHED).
For delete: `listingDelete` (removes entirely -- use for hard delist).


### 2.5 Implement: `fetchListings(account, cursor?)`

**Behavior:**
1. If `account.accessToken` is falsy, return empty `PaginatedListings` (same as eBay pattern).
2. Load config via `loadWhatnotConfig()`.
3. Execute `FetchListings` GraphQL query with `first: 50` (batch size from Lister Canonical Section 6.2) and `after: cursor` (or null for first page).
4. On GraphQL errors, log and return empty result.
5. On HTTP 401, log warning and return empty (token expired -- caller handles refresh).
6. For each node in `listings.nodes`:
   a. Validate with `WhatnotListingSchema.safeParse()`.
   b. Skip invalid items (log warning).
   c. Normalize via `normalizeWhatnotListing()` then `toExternalListing()`.
   d. Filter to only `ACTIVE` status (same as eBay pattern).
7. Return `PaginatedListings` with `cursor: pageInfo.endCursor`, `hasMore: pageInfo.hasNextPage`, `totalEstimate: null` (GraphQL does not return total count).

### 2.6 Implement: `fetchSingleListing(account, externalId)`

**Behavior:**
1. If `account.accessToken` is falsy, throw `Error('No access token')`.
2. Load config, execute `FetchListing` query with `id: externalId`.
3. On GraphQL errors or null data, throw `Error('Whatnot listing not found')`.
4. Validate with `WhatnotListingSchema.safeParse()`. If invalid, throw with validation error.
5. Normalize and return as `ExternalListing`.

### 2.7 Implement: `createListing(account, listing)`

**Behavior (2-step: create + publish):**
1. If `account.accessToken` is falsy, return `{ success: false, externalId: null, externalUrl: null, error: 'No credentials', retryable: false }`.
2. Load config.
3. Transform `TransformedListing` to Whatnot input:
   - `title`: `listing.title` (truncated to 200 chars per capabilities.maxTitleLength).
   - `description`: `listing.description` (truncated to 5000 chars per capabilities.maxDescriptionLength).
   - `price`: `{ amount: (listing.priceCents / 100).toFixed(2), currencyCode: 'USD' }` -- convert cents to decimal string.
   - `media`: `listing.images.sort((a,b) => a.sortOrder - b.sortOrder).map(i => ({ url: i.url }))`, max 10 images per capabilities.maxImagesPerListing.
   - `quantity`: `listing.quantity`.
   - `productTaxonomyNodeId`: `listing.category?.externalCategoryId` if available.
4. Execute `listingCreate` mutation.
5. Check `userErrors` array -- if non-empty, return failure with first error message, `retryable: false`.
6. Extract `listing.id` from response.
7. Execute `listingPublish` mutation with the new listing ID.
8. Check `userErrors` on publish -- if non-empty, return partial success info (listing created but not published): `{ success: false, externalId: listingId, externalUrl: null, error: 'Created but publish failed: ' + errorMsg, retryable: true }`.
9. On full success: return `{ success: true, externalId: listingId, externalUrl: 'https://www.whatnot.com/listings/' + listingId, retryable: false }`.
10. On network errors: return `{ success: false, externalId: null, externalUrl: null, error: String(err), retryable: true }`.
11. On HTTP 429/5xx: `retryable: true`. On 4xx (non-429): `retryable: false`.

### 2.8 Implement: `updateListing(account, externalId, changes)`

**Behavior:**
1. If `account.accessToken` is falsy, return `{ success: false, error: 'No credentials', retryable: false }`.
2. Load config.
3. Build partial input object from `changes` (only include fields that are present):
   - `changes.title` -> `title`
   - `changes.description` -> `description`
   - `changes.priceCents` -> `price: { amount: (changes.priceCents / 100).toFixed(2), currencyCode: 'USD' }`
   - `changes.images` -> `media` (sorted, mapped, max 10)
   - `changes.quantity` -> `quantity`
   - `changes.category` -> `productTaxonomyNodeId`
4. Execute `listingUpdate` mutation with `id: externalId` and `input`.
5. Check `userErrors` -- if non-empty, return failure.
6. On success: return `{ success: true, retryable: false }`.
7. On HTTP 429/5xx: `retryable: true`.

### 2.9 Implement: `delistListing(account, externalId)`

**Behavior:**
1. If `account.accessToken` is falsy, return `{ success: false, error: 'No credentials', retryable: false }`.
2. Load config.
3. Execute `listingUnpublish` mutation with `id: externalId`.
4. Check `userErrors` -- if non-empty AND the error indicates listing is already unpublished/deleted, treat as success (idempotent).
5. On success: return `{ success: true, retryable: false }`.
6. On HTTP 404 or "not found" GraphQL error: return success (listing already gone -- idempotent, same as eBay 404 pattern).
7. On other errors: `{ success: false, error: message, retryable: status >= 500 || status === 429 }`.

### 2.10 Implement: `verifyListing(account, externalId)`

**Behavior:**
1. If `account.accessToken` is falsy, return the current stub response (exists: false, status: UNKNOWN).
2. Load config.
3. Execute `FetchListing` query for the given `externalId`.
4. If GraphQL returns null/errors: return `{ exists: false, status: 'REMOVED', ... }`.
5. If listing found:
   - `exists: true`
   - `status`: map Whatnot status to VerificationResult status (PUBLISHED->ACTIVE, UNPUBLISHED->ENDED, SOLD->SOLD)
   - `priceCents`: parse from Money type via `parseMoneyToCents()`
   - `quantity`: from product variants if available, else 1
   - `lastModifiedAt`: parse `updatedAt` timestamp
   - `diff: null` (diff computation is done by caller, not connector)

```typescript
// src/lib/crosslister/connectors/whatnot-transform.ts

export function toWhatnotInput(listing: TransformedListing): WhatnotListingInput {
  return {
    title: listing.title.slice(0, 200),  // maxTitleLength
    description: listing.description.slice(0, 5000),  // maxDescriptionLength
    price: {
      amount: (listing.priceCents / 100).toFixed(2),
      currencyCode: 'USD',
    },
    media: listing.images
      .sort((a, b) => a.sortOrder - b.sortOrder)
      .slice(0, 10)  // maxImagesPerListing
      .map((img) => ({ url: img.url })),
    quantity: listing.quantity,
    productTaxonomyNodeId: listing.category?.externalCategoryId || undefined,
    condition: listing.condition || undefined,
  };
}

export function toWhatnotPartialInput(
  changes: Partial<TransformedListing>,
): Partial<WhatnotListingInput> {
  const input: Partial<WhatnotListingInput> = {};
  if (changes.title !== undefined) input.title = changes.title.slice(0, 200);
  if (changes.description !== undefined) input.description = changes.description.slice(0, 5000);
  if (changes.priceCents !== undefined) {
    input.price = { amount: (changes.priceCents / 100).toFixed(2), currencyCode: 'USD' };
  }
  if (changes.images !== undefined) {
    input.media = changes.images
      .sort((a, b) => a.sortOrder - b.sortOrder)
      .slice(0, 10)
      .map((img) => ({ url: img.url }));
  }
  if (changes.quantity !== undefined) input.quantity = changes.quantity;
  if (changes.category !== undefined) {
    input.productTaxonomyNodeId = changes.category?.externalCategoryId || undefined;
  }
  return input;
}
```

## 3. CONSTRAINTS -- WHAT NOT TO DO

### Code Constraints
- **No `as any`** -- all GraphQL responses must be properly typed. Use the response interfaces defined in Section 2.11.
- **No GraphQL client libraries** (e.g., no `graphql-request`, no `@apollo/client`). Use plain `fetch()` with JSON body -- this is the established H2.1 pattern.
- **Integer cents only** -- all price conversions via `parseMoneyToCents()` (inbound) and `(priceCents / 100).toFixed(2)` (outbound). Never `parseFloat` on money except in the existing `parseMoneyToCents()` helper.
- **Max 300 lines per NEW file** -- the existing `whatnot-connector.ts` is already 362 lines (pre-existing violation accepted per user preference), but new helper files (`whatnot-graphql.ts`, `whatnot-transform.ts`) must stay under 300 lines.
- **No hardcoded fee rates** -- the platform-fees.ts mapping already handles WHATNOT. Connector methods never reference fee rates.
- **No `console.log`** -- use `logger.info/warn/error` with `[WhatnotConnector]` or `[WhatnotGraphQL]` prefix.

### Business Logic Constraints
- **BIN listings only** -- Whatnot also has live-selling/auction formats. This connector ONLY handles BIN (fixed-price, buy-it-now) listings. Do NOT implement any live-selling or auction mutations.
- **No livestream assignment** -- The `listingAssignToLivestream` and `listingRemoveFromLivestream` mutations exist in the API but are OUT OF SCOPE for H2.2.
- **Imports are always free** -- The fetchListings method is used by the import pipeline. It must not check/enforce any ListerTier limits (the import pipeline caller handles that).
- **Syncs do NOT count as publishes** -- updateListing is used for sync operations. Connector never meters publishes.

### API Constraints
- **Environment-aware URLs** -- always use `getBaseUrl()` / `getGraphQLUrl()` to resolve production vs staging URLs. Never hardcode `api.whatnot.com`.
- **Refresh token invalidation** -- Whatnot invalidates used refresh tokens. If any method gets a 401, it must NOT attempt to refresh the token itself -- it returns an error and lets the caller (publish-service/import-pipeline) handle token refresh via `refreshAuth()`.
- **Rate limit awareness** -- on HTTP 429, always set `retryable: true`. The scheduler handles retry timing.
- **Batch size: 50** -- per Lister Canonical Section 6.2, import fetches use 50 items per batch. The `first` variable in the listings query must be 50.

### Whatnot-Specific Gotchas
- **Money type is `{ amount: string, currencyCode: string }`** -- `amount` is a decimal string like `"12.99"`, NOT an integer. Must convert to/from integer cents.
- **Status mapping**: PUBLISHED = ACTIVE, UNPUBLISHED = ENDED, SOLD = SOLD. Unknown statuses default to ENDED.
- **Media filter**: Whatnot media array may contain VIDEO type entries. Only IMAGE type entries should be used for `ExternalImage[]`.
- **`userErrors` pattern**: Whatnot mutations return success HTTP 200 even on validation failures. Must ALWAYS check the `userErrors` array in mutation responses. An empty `userErrors` array means success.
- **Listing URL pattern**: `https://www.whatnot.com/listings/{id}` -- already established in normalizer.
