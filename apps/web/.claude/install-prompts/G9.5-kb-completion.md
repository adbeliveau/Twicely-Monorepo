### 2.5 Queries

**Add `searchKbArticles` to `src/lib/queries/kb-articles.ts`:**

```typescript
export async function searchKbArticles(
  query: string,
  userAudience: 'ALL' | 'BUYER' | 'SELLER' | null,
  categorySlug?: string,
  limit?: number
): Promise<KbArticleListItem[]>
```

Implementation:
- Build audience filter using existing `buildAudienceFilter` helper
- Filter: `status = 'PUBLISHED'` AND `audience IN (audienceValues)`
- Text search: split `query` into words, for each word require it appears in title, excerpt, body, or tags (using `ilike` + `or`, same pattern as `src/lib/search/listings.ts`)
- If `categorySlug` provided: join on `kbCategory` to filter by slug
- Order by: title matches first (boost title matches), then by `viewCount` desc
- Limit: `limit ?? 20`
- This uses DB-based search, consistent with the existing `searchListings()` pattern. Typesense indexing for KB articles is defined in the Schema Doc (section 26, `kb_articles` index) but Typesense client is not yet wired (the listing search also uses DB). A future step can add Typesense when it's wired.

## 4. ACCEPTANCE CRITERIA

### Functional
1. **updateKbArticle works**: Staff with HELPDESK_LEAD+ role can update any field on an existing article. Changes persist in DB. `updatedAt` is set.
2. **Editor routes to correct action**: New article form calls `createKbArticle`. Edit article form (with `initialData.id`) calls `updateKbArticle`.
3. **Status transitions work**: DRAFT -> REVIEW via "Submit for Review" button. DRAFT/REVIEW -> PUBLISHED via "Publish" button (MANAGER+ only). PUBLISHED -> ARCHIVED via "Archive" button. All transitions revalidate paths.
4. **Publish gating**: HELPDESK_LEAD cannot publish (returns access denied). HELPDESK_MANAGER and ADMIN can publish. The CASL `manage` rule allows leads to create/update/archive, but publish requires explicit MANAGER+ check.
5. **Category CRUD complete**: Create, update, delete, reorder all work. Delete fails if category has articles. Edit dialog pre-fills all fields.
6. **Help center search functional**: Typing in the search input on `/h` returns real results from DB. Results show title, excerpt, category. Clicking a result navigates to the article. Search respects audience gating.
7. **Search results page**: Visiting `/h?q=nike` shows search results server-side.
8. **Markdown rendering**: Article bodies render with proper Markdown formatting (headings, bold, italic, links, lists, code blocks, tables). No raw Markdown syntax visible to users.
9. **searchKeywords field**: The editor exposes a searchKeywords input. Values are saved and used in search.

### Negative

11. **HELPDESK_AGENT cannot create, update, or delete articles or categories** — returns "Access denied"
12. **Cannot delete category with articles** — returns descriptive error

14. **Non-PUBLISHED articles do not appear in public help center** — search and browse both filter by status
