## 1. FEATURE OVERVIEW: TheRealReal Consignment Model

TheRealReal is a **luxury consignment marketplace**. Unlike standard listing marketplaces (Poshmark, FB Marketplace), sellers do NOT set prices or handle sales directly. The flow is:

1. **Consignment submission:** Seller submits items to TheRealReal for consignment. TRR authenticates luxury items and sets the selling price.
2. **TRR manages the listing:** TRR photographs, describes, prices, and lists the item. Sellers can view their consigned items in "My Consignments" but cannot edit the listing freely.
3. **TRR handles the sale:** When an item sells, TRR takes a commission (typically 20-40%) and pays out the remainder to the consignor.

**Impact on this content script:**

| Capability | Poshmark (H1.2) | FB Marketplace (H1.2) | TheRealReal (H1.3) |
|---|---|---|---|
| Session capture | JWT + username | c_user cookie (login confirmation) | sessionId + csrfToken + userId + email |
| Listing scrape | Full listing detail (`/listing/<id>`) | Full listing detail (`/marketplace/item/<id>`) | Consignment items from "My Consignments" page (`/account/consignments`) |
| Auto-fill | Full listing creation form | Full listing creation form | Limited: consignment submission form (title, brand, category, condition, description, photos NOT automated) |

**Key differences from H1.2 platforms:**
- TheRealReal uses a traditional server-rendered site with React hydration -- NO `__NEXT_DATA__` like Poshmark
- The internal API at `https://www.therealreal.com/api/v1` is used by the server-side connector. The content script scrapes the DOM or intercepts page JSON embedded in `<script>` tags
- Session cookies include `session_id` and CSRF tokens in `<meta>` tags (standard Rails CSRF pattern)
- TRR sets the price, so `priceCents` may be labeled "Estimated Value" or "List Price" rather than a seller-set price
- The "My Consignments" page shows items the seller has consigned, each with: title, brand/designer, price, condition grade (Excellent/Very Good/Good/Fair/Poor), authentication status, and TRR-assigned item ID
- Auto-fill scope is narrower than Poshmark/FB because TRR's consignment form has fewer seller-editable fields

#### 3.1.1 Session Capture

TheRealReal uses a standard Rails-style session architecture:
- Session ID in a cookie named `session_id` (or `_session_id` or `_therealreal_session`)
- CSRF token in a `<meta name="csrf-token">` tag in the page `<head>`
- User data may be embedded in a `<script>` tag containing JSON (e.g., `window.__TRR_USER__` or `window.__INITIAL_STATE__`) OR available from DOM elements in the account header

**Implementation strategy:**
```typescript
export async function captureSession(): Promise<Record<string, unknown> | null> {
  // 1. Get session cookie -- try multiple known cookie names
  const sessionId =
    getCookieValue('session_id') ??
    getCookieValue('_session_id') ??
    getCookieValue('_therealreal_session');
  if (!sessionId) return null; // Not logged in

  // 2. Get CSRF token from meta tag (standard Rails pattern)
  const csrfMeta = document.querySelector<HTMLMetaElement>('meta[name="csrf-token"]');
  const csrfToken = csrfMeta?.content ?? '';

  // 3. Get user info -- Strategy A: embedded JSON in script tags
  const userData = extractUserFromPageScripts();
  if (userData) {
    return {
      sessionId,
      csrfToken,
      userId: userData.userId,
      email: userData.email,
    };
  }

  // 4. Strategy B: DOM selectors for user info (account dropdown, profile link)
  const userFromDom = extractUserFromDom();
  if (userFromDom) {
    return {
      sessionId,
      csrfToken,
      userId: userFromDom.userId,
      email: userFromDom.email,
    };
  }

  // 5. If we have a session cookie and CSRF but cannot identify user, still report
  //    partial session (server can validate by making a test API call)
  if (csrfToken) {
    return {
      sessionId,
      csrfToken,
      userId: '',
      email: '',
    };
  }

  return null;
}
```

**Helper: `extractUserFromPageScripts()`**
TheRealReal may embed user data in inline scripts. Search `<script>` tags for patterns like:
- `window.__TRR_USER__` or `window.__INITIAL_STATE__` or `window.__trr`
- JSON containing fields like `id`, `email`, `first_name`, `last_name`

```typescript
function extractUserFromPageScripts(): { userId: string; email: string } | null {
  const scripts = document.querySelectorAll<HTMLScriptElement>('script:not([src])');
  for (const script of scripts) {
    const text = script.textContent ?? '';

    // Look for embedded user JSON patterns
    const patterns = [
      /window\.__TRR_USER__\s*=\s*({.*?});/s,
      /window\.__INITIAL_STATE__\s*=\s*({.*?});/s,
      /"currentUser"\s*:\s*({.*?})/s,
      /"user"\s*:\s*({.*?"email".*?})/s,
    ];

    for (const pattern of patterns) {
      const match = text.match(pattern);
      if (match?.[1]) {
        try {
          const data = JSON.parse(match[1]) as Record<string, unknown>;
          const userId = String(data['id'] ?? data['user_id'] ?? '');
          const email = String(data['email'] ?? '');
          if (userId || email) {
            return { userId, email };
          }
        } catch {
          // Parse failed, try next pattern
        }
      }
    }
  }
  return null;
}
```

**Helper: `extractUserFromDom()`**
Fall back to reading user info from the DOM:
- Account link: `a[href="/account"]`, `a[href*="/my-account"]`
- Email: may be in an account dropdown or profile section
- User ID: may be in a data attribute

```typescript
function extractUserFromDom(): { userId: string; email: string } | null {
  // Check account link for user name
  const accountLink =
    document.querySelector<HTMLAnchorElement>('a[href="/account"]') ??
    document.querySelector<HTMLAnchorElement>('a[href*="/account/profile"]');

  // Check for email in account dropdown
  const emailEl =
    document.querySelector<HTMLElement>('[data-email]') ??
    document.querySelector<HTMLElement>('.account-email');
  const email = emailEl?.getAttribute('data-email') ?? emailEl?.textContent?.trim() ?? '';

  // Check for user ID in data attributes
  const userIdEl = document.querySelector<HTMLElement>('[data-user-id]');
  const userId = userIdEl?.getAttribute('data-user-id') ?? '';

  if (!email && !userId) return null;
  return { userId, email };
}
```

#### 3.1.2 Listing Scrape (Consignment Page)

TheRealReal consignors view their items at `/account/consignments` or `/account/consignments/<id>`. Individual consignment items may also be viewable at `/products/<id>`.

**Important distinction:** On TRR, we scrape from TWO page types:
1. **Product detail page** (`/products/<slug-or-id>`): Public listing page with title, price, brand, images, condition
2. **My Consignments page** (`/account/consignments` or `/account/consignments/<id>`): Seller's private view of their consigned items

The scrape should work on BOTH page types.

```typescript
export function scrapeListing(): ScrapedListing | null {
  const pathname = window.location.pathname;

  // Product detail page: /products/<slug-or-id>
  if (pathname.startsWith('/products/')) {
    return scrapeProductDetailPage();
  }

  // Individual consignment page: /account/consignments/<id>
  const consignmentMatch = pathname.match(/^\/account\/consignments\/([^/?#]+)/);
  if (consignmentMatch?.[1]) {
    return scrapeConsignmentDetailPage(consignmentMatch[1]);
  }

  // Not on a scrapable page
  return null;
}
```

1. Try to find JSON-LD structured data (`<script type="application/ld+json">`) which luxury sites often embed for SEO
2. Fall back to DOM selectors

```typescript
function scrapeProductDetailPage(): ScrapedListing | null {
  // Extract external ID from URL: /products/<slug>
  // TRR URLs are like: /products/chanel-classic-flap-bag-123456
  const pathParts = window.location.pathname.split('/');
  const slug = pathParts[pathParts.length - 1] ?? null;
  if (!slug) return null;

  // Strategy 1: JSON-LD (most reliable for TRR product pages)
  const listing = scrapeFromJsonLd(slug);
  if (listing) return listing;

  // Strategy 2: DOM selectors
  return scrapeProductFromDom(slug);
}
```

**JSON-LD extraction:**
```typescript
function scrapeFromJsonLd(externalId: string): ScrapedListing | null {
  const ldScripts = document.querySelectorAll<HTMLScriptElement>(
    'script[type="application/ld+json"]',
  );
  for (const script of ldScripts) {
    try {
      const data = JSON.parse(script.textContent ?? '') as Record<string, unknown>;
      // Look for Product schema
      if (data['@type'] !== 'Product') continue;

      const title = typeof data['name'] === 'string' ? data['name'].trim() : null;
      if (!title) continue;

      // Price from offers
      let priceCents: number | null = null;
      const offers = data['offers'];
      if (offers && typeof offers === 'object') {
        const o = offers as Record<string, unknown>;
        const price = o['price'];
        if (typeof price === 'string') {
          priceCents = parsePriceStringToCents(price);
        } else if (typeof price === 'number') {
          priceCents = Math.round(price * 100);
        }
      }
      if (priceCents === null) continue;

      // Description
      const description = typeof data['description'] === 'string'
        ? data['description'].trim()
        : '';

      // Brand
      let brand: string | null = null;
      const brandObj = data['brand'];
      if (brandObj && typeof brandObj === 'object') {
        const b = brandObj as Record<string, unknown>;
        brand = typeof b['name'] === 'string' ? b['name'] : null;
      }

      // Images
      const imageUrls = extractImagesFromJsonLd(data);

      // Condition from DOM (JSON-LD typically does not include TRR condition grade)
      const condition = extractTrrConditionFromDom();

      return {
        externalId,
        title,
        priceCents,
        description,
        condition,
        brand,
        category: extractTrrCategoryFromDom(),
        size: extractTrrSizeFromDom(),
        imageUrls,
        url: window.location.href,
      };
    } catch {
      // Parse failed, try next script
    }
  }
  return null;
}
```

```typescript
function extractTrrConditionFromDom(): string | null {
  // TRR condition grades: Excellent, Very Good, Good, Fair, Poor
  const condEl =
    document.querySelector<HTMLElement>('[data-testid="condition-grade"]') ??
    document.querySelector<HTMLElement>('.condition-grade') ??
    document.querySelector<HTMLElement>('.item-condition');
  if (condEl) return condEl.textContent?.trim() ?? null;

  // Search for known condition text strings in the page
  const known = ['Excellent', 'Very Good', 'Good', 'Fair', 'Poor'];
  const allEls = document.querySelectorAll<HTMLElement>('span, div, p');
  for (const el of allEls) {
    const text = el.textContent?.trim() ?? '';
    if (known.includes(text) && el.childElementCount === 0) return text;
  }
  return null;
}

function extractTrrBrandFromDom(): string | null {
  const brandEl =
    document.querySelector<HTMLElement>('[data-testid="product-designer"]') ??
    document.querySelector<HTMLElement>('.product-designer') ??
    document.querySelector<HTMLElement>('[itemprop="brand"]') ??
    document.querySelector<HTMLElement>('.designer-name');
  return brandEl?.textContent?.trim() ?? null;
}

function extractTrrCategoryFromDom(): string | null {
  // Breadcrumb navigation
  const breadcrumbs = Array.from(
    document.querySelectorAll<HTMLElement>('nav[aria-label="breadcrumb"] a, .breadcrumb a, .breadcrumbs a'),
  );
  if (breadcrumbs.length > 0) {
    return breadcrumbs
      .map((el) => el.textContent?.trim())
      .filter(Boolean)
      .join(' > ');
  }
  return null;
}

function extractTrrSizeFromDom(): string | null {
  const sizeEl =
    document.querySelector<HTMLElement>('[data-testid="product-size"]') ??
    document.querySelector<HTMLElement>('.product-size') ??
    document.querySelector<HTMLElement>('.item-size');
  return sizeEl?.textContent?.trim() ?? null;
}

function extractTrrImagesFromDom(): string[] {
  // TRR product images in gallery/carousel
  const imgs = Array.from(
    document.querySelectorAll<HTMLImageElement>(
      '.product-gallery img, .product-images img, [data-testid="product-image"] img, .carousel img',
    ),
  );

  const urls = imgs
    .map((img) => {
      // Prefer srcset for highest resolution
      if (img.srcset) {
        const parts = img.srcset.split(',').map((s) => s.trim().split(' ')[0]).filter(Boolean);
        if (parts.length > 0) return parts[parts.length - 1];
      }
      return img.src || null;
    })
    .filter((url): url is string => {
      return typeof url === 'string' && url.startsWith('http') && !url.includes('placeholder');
    });

  // Deduplicate
  return [...new Set(urls)];
}

function extractImagesFromJsonLd(data: Record<string, unknown>): string[] {
  const image = data['image'];
  if (typeof image === 'string') return [image];
  if (Array.isArray(image)) {
    return image
      .map((i) => {
        if (typeof i === 'string') return i;
        if (i && typeof i === 'object') {
          const obj = i as Record<string, unknown>;
          return typeof obj['url'] === 'string' ? obj['url'] : null;
        }
        return null;
      })
      .filter((u): u is string => typeof u === 'string' && u.startsWith('http'));
  }
  return [];
}
```

#### 3.1.3 Auto-fill (Consignment Submission Form)

TheRealReal's consignment submission form is at a URL like `/consign` or `/account/consignment/new` or similar. The form is more limited than Poshmark/FB because TRR handles much of the listing process themselves.

Fields the content script can fill:
- **Title / Item name**: text input for the item name
- **Brand / Designer**: text input or searchable dropdown for designer name
- **Category**: dropdown or multi-level category selector
- **Condition**: radio buttons or dropdown (Excellent / Very Good / Good / Fair / Poor)
- **Description / Notes**: textarea for seller notes about the item
- **Size**: text input or size selector

Fields the content script CANNOT fill:
- **Photos**: browser security prevents programmatic file input
- **Price**: TRR sets the price, NOT the consignor (this is the key consignment model difference)

```typescript
export async function autofillListing(payload: AutofillPayload): Promise<boolean> {
  let allFilled = true;

  // Title / Item Name
  const titleInput = document.querySelector<HTMLInputElement>(
    'input[name="title"], input[name="item_name"], input[placeholder*="Item"], input[placeholder*="Title"], input[data-testid="consignment-title"]',
  );
  if (titleInput) {
    setReactInputValue(titleInput, payload.title);
    await randomDelay(300, 800);
  } else {
    allFilled = false;
  }

  // Description / Notes
  const descInput = document.querySelector<HTMLTextAreaElement>(
    'textarea[name="description"], textarea[name="notes"], textarea[placeholder*="Description"], textarea[data-testid="consignment-description"]',
  );
  if (descInput) {
    setReactInputValue(descInput, payload.description);
    await randomDelay(300, 800);
  } else {
    // Description not required -- do not fail
  }

  // Brand / Designer
  if (payload.brand) {
    const brandInput = document.querySelector<HTMLInputElement>(
      'input[name="designer"], input[name="brand"], input[placeholder*="Designer"], input[placeholder*="Brand"], input[data-testid="consignment-designer"]',
    );
    if (brandInput) {
      setReactInputValue(brandInput, payload.brand);
      await randomDelay(500, 1200); // Longer delay for autocomplete dropdown
    }
  }

  // Size (optional)
  if (payload.size) {
    const sizeInput = document.querySelector<HTMLInputElement>(
      'input[name="size"], input[placeholder*="Size"], select[name="size"], input[data-testid="consignment-size"]',
    );
    if (sizeInput) {
      if (sizeInput.tagName === 'SELECT') {
        // Select element: find matching option
        const options = Array.from((sizeInput as unknown as HTMLSelectElement).options);
        const match = options.find(
          (opt) => opt.text.trim().toLowerCase() === payload.size?.toLowerCase(),
        );
        if (match) {
          (sizeInput as unknown as HTMLSelectElement).value = match.value;
          sizeInput.dispatchEvent(new Event('change', { bubbles: true }));
          await randomDelay(200, 500);
        }
      } else {
        setReactInputValue(sizeInput, payload.size);
        await randomDelay(300, 800);
      }
    }
  }

  // Condition (optional -- click matching grade button/radio)
  if (payload.condition) {
    const condButtons = Array.from(
      document.querySelectorAll<HTMLElement>(
        '[data-testid*="condition"], input[name="condition"], [role="radio"], .condition-option',
      ),
    );
    const matchingCond = condButtons.find(
      (btn) => btn.textContent?.trim().toLowerCase() === payload.condition?.toLowerCase(),
    );
    if (matchingCond) {
      matchingCond.click();
      await randomDelay(200, 500);
    }
  }

  // NOTE: Price is NOT filled -- TRR sets the price, not the consignor.
  // NOTE: Photos are NOT filled -- browser security restriction.

  return allFilled;
}
```

## 4. CONSTRAINTS -- WHAT NOT TO DO

### Banned Patterns
- Do NOT make direct API calls from the content script to the Twicely backend or to TheRealReal's API. ALL communication goes through `chrome.runtime.sendMessage` to the service worker, which then uses `apiClient`
- Do NOT use `eval()`, `new Function()`, or `innerHTML` with untrusted content (MV3 CSP violation)
- Do NOT store passwords, API keys, or session tokens in the content script or `window` scope. Session data flows: DOM/cookies read -> `chrome.runtime.sendMessage` -> service worker -> Twicely API
- Do NOT use `document.write` or synchronous XHR
- Do NOT import any Node.js modules or modules that transitively depend on Node.js (postgres, fs, etc.)
- Do NOT use `as any`, `@ts-ignore`, or `@ts-expect-error`
- Do NOT use `console.log` -- use `console.debug` if debugging is needed (will be stripped in production)
- Do NOT attempt to fill the price field during auto-fill -- TRR sets prices, not consignors
- Do NOT use `parseNextData()` from shared-utils -- TRR is NOT a Next.js app

### Platform-Specific Gotchas
- **TRR is NOT Next.js:** Unlike Poshmark, TRR does not have `__NEXT_DATA__`. It uses a traditional server-rendered site (likely Rails) with React hydration. Session data is in cookies and meta tags, not JS globals
- **CSRF token is required:** The server-side connector uses the CSRF token for all API calls. If the content script cannot extract it from `<meta name="csrf-token">`, the session data is incomplete
- **Cookie names may vary:** TRR may use `session_id`, `_session_id`, or `_therealreal_session`. Try all three
- **Consignment vs product pages:** The listing scrape must handle both `/products/<slug>` (public) and `/account/consignments/<id>` (private) page structures
- **Price is set by TRR:** The "price" shown on a consignment is what TRR has priced the item at, not what the seller requested. Still scrape it as `priceCents` -- it is the listing price
- **Authentication status:** TRR has an authentication_status field (authenticated/pending/not_required) for luxury items. This is captured by the condition/itemSpecifics in the server-side normalizer but is NOT part of `ScrapedListing`. It is OK to omit it from the content script scrape
- **Randomized delays:** Per Lister Canonical Section 9.4, all Tier C interactions should include human-like random delays (2-8 seconds between actions). The auto-fill already uses `randomDelay()` between field fills

### Vocabulary Enforcement
- `ScrapedListing.priceCents` -- integer cents, never dollars as float
- No banned terms from CLAUDE.md (SellerTier, FVF, wallet, etc.) -- though these are unlikely in extension code

## 10. SPEC GAPS AND OWNER DECISIONS NEEDED

The following items are NOT fully specified in canonical documents. The prompt provides best-guess implementations but the installer should flag these:

1. **TRR cookie name:** The exact session cookie name (`session_id`, `_session_id`, or `_therealreal_session`) depends on TRR's current implementation. The content script tries all three. This is inherently fragile and expected to need updates as TRR deploys changes.

2. **TRR user data embedding:** The exact pattern for how TRR embeds user data in page scripts (`window.__TRR_USER__`, `window.__INITIAL_STATE__`, etc.) is not documented. The content script uses pattern matching across multiple known patterns. If none match, it falls back to DOM extraction and then partial session reporting.

3. **Consignment submission form URL:** The exact URL and form structure for TRR's consignment submission (`/consign`, `/account/consignment/new`, etc.) is not documented in the canonical specs. The auto-fill uses generic form field selectors that should work regardless of the exact URL.

4. **Lister Canonical has no dedicated TRR section:** Unlike Poshmark (Section 16 with detailed 3-mode automation), there is no dedicated TheRealReal section in the Lister Canonical. TRR is referenced as a Tier C connector alongside Poshmark in Section 9.4 and in the rollout timeline (Section 27.2: Month 5-6). The content script implementation is based on the general Tier C patterns and the consignment model's natural constraints.

5. **TRR automation status (partial):** The rollout table shows TheRealReal automation as "partial" (yellow circle). This aligns with the consignment model -- full automation (like Poshmark's sharing/relisting) makes less sense when TRR controls the listing. The auto-fill covers consignment submission, which is the primary automation-relevant action.
