#### 2a. P&L Report Data

```typescript
export interface PnlReportData {
  periodStart: string;  // ISO date
  periodEnd: string;    // ISO date
  generatedAt: string;  // ISO timestamp

  // Revenue section
  grossRevenueCents: number;         // sum of order.totalCents (COMPLETED in period)
  totalOrderCount: number;

  // COGS section
  cogsTotalCents: number;            // sum of listing.cogsCents for sold items
  grossProfitCents: number;          // revenue - COGS

  // Fee section (platform fees)
  tfFeesCents: number;              // ORDER_TF_FEE ledger entries
  stripeFeesCents: number;          // ORDER_STRIPE_PROCESSING_FEE
  boostFeesCents: number;           // ORDER_BOOST_FEE
  insertionFeesCents: number;       // INSERTION_FEE
  localFeesCents: number;           // LOCAL_TRANSACTION_FEE
  authFeesCents: number;            // AUTH_FEE_SELLER + AUTH_FEE_BUYER
  subscriptionChargesCents: number;  // SUBSCRIPTION_CHARGE + FINANCE_SUBSCRIPTION_CHARGE
  totalPlatformFeesCents: number;

  // Shipping section
  shippingCostsCents: number;       // SHIPPING_LABEL_PURCHASE

  // Net after fees
  netAfterFeesCents: number;        // grossProfit - platformFees - shipping

  // Operating expenses (manual)
  operatingExpensesCents: number;    // sum of expense table
  expensesByCategory: Array<{ category: string; totalCents: number; count: number }>;

  // Mileage deductions
  mileageDeductionCents: number;    // sum of mileageEntry.deductionCents
  totalMiles: number;
  tripCount: number;

  // Net profit
  netProfitCents: number;           // netAfterFees - operatingExpenses - mileageDeductions

  // KPIs
  avgSalePriceCents: number;
  effectiveFeeRatePercent: number;  // totalPlatformFees / grossRevenue * 100
  cogsMarginPercent: number;        // grossProfit / grossRevenue * 100 (0 if no COGS)
}
```

Function signature: `getPnlReportData(userId: string, periodStart: Date, periodEnd: Date): Promise<PnlReportData>`

This is a more comprehensive version of `getFinanceDashboardKPIs` that uses **exact date range** (not "days ago") and includes ALL fee types (insertion fees, local fees, auth fees, subscription charges) that the 30-day dashboard omits.

Implementation approach:
1. Query COMPLETED orders in `[periodStart, periodEnd]` for revenue
2. Query ledger entries by type in period for each fee category
3. Query expenses in period for operating expenses + category breakdown
4. Query mileage entries in period for deductions
5. Query COGS via order -> orderItem -> listing join (same pattern as `getCogsSummary`)
6. Calculate all derived fields using integer cents arithmetic

#### 2b. Balance Sheet Data

```typescript
export interface BalanceSheetData {
  periodStart: string;
  periodEnd: string;
  generatedAt: string;

  // Assets
  assets: {
    // Current Assets
    availableForPayoutCents: number;   // sellerBalance.availableCents (current)
    pendingCents: number;              // sellerBalance.pendingCents (in escrow)
    inventoryValueCents: number;       // sum of priceCents for ACTIVE listings
    inventoryCount: number;            // count of ACTIVE listings
    totalCurrentAssetsCents: number;
  };

  // Liabilities
  liabilities: {
    reservedCents: number;             // sellerBalance.reservedCents (holds)
    pendingRefundsCents: number;       // sum of orders in REFUND_PENDING status (if applicable)
    totalLiabilitiesCents: number;
  };

  // Equity (owner's equity = assets - liabilities)
  equity: {
    netEquityCents: number;
    // Period net profit from P&L
    periodNetProfitCents: number;
    totalEquityCents: number;
  };
}
```

Function signature: `getBalanceSheetData(userId: string, periodStart: Date, periodEnd: Date): Promise<BalanceSheetData>`

Implementation:
1. Query `sellerBalance` for current cash position (available, pending, reserved)
2. Query active listings for inventory value: `sum(priceCents) WHERE status = 'ACTIVE' AND sellerId = userId`
3. Reserved cents as liability
4. Calculate net equity as totalAssets - totalLiabilities
5. Include period net profit from `getPnlReportData()` result (pass as parameter or call inline)

**NOTE**: Balance sheet is a **point-in-time snapshot**, not a period aggregate. The `periodEnd` date is the "as of" date. `periodStart` provides context for the period P&L included in equity. For a resale business operating without formal accounting, this is a simplified cash-basis balance sheet, not a full accrual-basis one. This is appropriate for the audience (individual resellers, not CPAs).

#### 2c. Cash Flow Statement Data

```typescript
export interface CashFlowData {
  periodStart: string;
  periodEnd: string;
  generatedAt: string;

  // Operating activities
  operating: {
    salesReceivedCents: number;        // ORDER_PAYMENT_CAPTURED entries
    refundsIssuedCents: number;        // REFUND_FULL + REFUND_PARTIAL (negative)
    platformFeesPaidCents: number;     // all fee-type ledger entries (negative)
    shippingCostsCents: number;        // SHIPPING_LABEL_PURCHASE (negative)
    operatingExpensesCents: number;    // manual expenses in period (negative)
    mileageDeductionCents: number;     // mileage deductions in period (negative)
    netOperatingCents: number;
  };

  // Financing activities (payouts)
  financing: {
    payoutsSentCents: number;          // PAYOUT_SENT entries (negative, money left platform)
    payoutsFailedReversedCents: number; // PAYOUT_FAILED + PAYOUT_REVERSED (positive)
    netFinancingCents: number;
  };

  // Summary
  netCashChangeCents: number;          // operating + financing
  beginningBalanceCents: number;       // sellerBalance snapshot at periodStart (approximated)
  endingBalanceCents: number;          // sellerBalance current
}
```

Function signature: `getCashFlowData(userId: string, periodStart: Date, periodEnd: Date): Promise<CashFlowData>`

Implementation:
1. Query ledger entries grouped by type group within `[periodStart, periodEnd]`
2. Operating: sales - refunds - fees - shipping - expenses - mileage
3. Financing: payouts sent (outflow) + payout reversals (inflow)
4. Beginning balance: `endingBalance - netCashChange` (derived approximation since we don't snapshot balances historically)
5. Ending balance: current `sellerBalance.availableCents + sellerBalance.pendingCents`

### 3. Report Generation Actions

**New file**: `src/lib/actions/finance-center-reports.ts`

Server actions for generating, listing, and downloading reports. Each action follows the established pattern in `finance-center-expenses.ts`.

#### 3a. generateReportAction

```typescript
export type GenerateReportResponse =
  | { success: true; report: SavedReport }
  | { success: false; error: string };

export async function generateReportAction(
  input: unknown,
): Promise<GenerateReportResponse>
```

Steps:
1. Parse input with `generateReportSchema` (see section 4)
2. `authorize()` + resolve userId (delegation-aware via `resolveUserId`)
3. Check `ability.can('create', sub('FinancialReport', { userId }))`
4. Check `getFinanceTier(userId) === 'PRO'`
5. Call appropriate data assembly function based on `reportType`:
   - `'PNL'` -> `getPnlReportData(userId, periodStart, periodEnd)`
   - `'BALANCE_SHEET'` -> `getBalanceSheetData(userId, periodStart, periodEnd)`
   - `'CASH_FLOW'` -> `getCashFlowData(userId, periodStart, periodEnd)`
6. Determine format:
   - `'JSON'` -> store snapshotJson directly, no fileUrl
   - `'CSV'` -> call `generateCsv(reportType, snapshotData)` -> upload to R2 -> set fileUrl
   - `'PDF'` -> call `generatePdfHtml(reportType, snapshotData)` -> upload to R2 as `.html` -> set fileUrl
7. Insert row into `financialReport` table with: userId, reportType, periodStart, periodEnd, snapshotJson, format, fileUrl
8. Return the saved report row
9. `revalidatePath('/my/selling/finances/statements')`

### 4. Validation Schemas

**File**: `src/lib/validations/finance-center.ts` (extend existing)

Add these schemas at the end of the file:

```typescript
export const REPORT_TYPES = ['PNL', 'BALANCE_SHEET', 'CASH_FLOW'] as const;
export const REPORT_FORMATS = ['JSON', 'CSV', 'PDF'] as const;

export const generateReportSchema = z
  .object({
    reportType: z.enum(REPORT_TYPES),
    periodStart: z.string().datetime(),
    periodEnd: z.string().datetime(),
    format: z.enum(REPORT_FORMATS).default('JSON'),
  })
  .strict()
  .refine(
    (data) => new Date(data.periodStart) < new Date(data.periodEnd),
    { message: 'Period start must be before period end', path: ['periodStart'] },
  );

export const listReportsSchema = z
  .object({
    page: z.number().int().positive().default(1),
    pageSize: z.number().int().positive().max(50).default(10),
    reportType: z.enum(REPORT_TYPES).optional(),
  })
  .strict();

export const deleteReportSchema = z
  .object({
    id: z.string().min(1),
  })
  .strict();

export type GenerateReportInput = z.infer<typeof generateReportSchema>;
export type ListReportsInput = z.infer<typeof listReportsSchema>;
export type DeleteReportInput = z.infer<typeof deleteReportSchema>;
```

**Note**: `TAX_PREP` and `INVENTORY_AGING` are valid values in the schema `reportType` column but are OUT OF SCOPE for D4.3. They will be added in later phases. The validation schema restricts to only the three types built in this step.

### 6. PDF (HTML) Export Generator

**New file**: `src/lib/finance/report-pdf.ts`

Generates self-contained HTML documents styled for printing. Users download the HTML and print to PDF via their browser, or the HTML file itself serves as the report.

**Rationale**: Puppeteer/wkhtmltopdf are not in the approved tech stack. React Email is for emails. The pragmatic approach is a print-optimized HTML document with `@media print` CSS. The `format` column stores `'PDF'` but the actual file is `.html` -- this is acceptable because the user experience is "download and print."

```typescript
export function generatePnlHtml(data: PnlReportData): string
export function generateBalanceSheetHtml(data: BalanceSheetData): string
export function generateCashFlowHtml(data: CashFlowData): string
```

Each function:
1. Returns a complete `<!DOCTYPE html>` document
2. Inline CSS (no external stylesheets) -- the file must be self-contained
3. `@media print` rules for clean printing (hide browser chrome, set margins)
4. Twicely branding: logo text "Twicely Financial Report", report type, period, seller info placeholder
5. Structured table layout matching the CSV sections
6. All monetary values formatted via `formatCentsToDollars()`
7. Footer: "Generated by Twicely Financial Center on [date]"
8. Max width 800px, serif font for formal document feel

### 7. Report Upload to R2

CSV and PDF (HTML) files are uploaded to R2 using the existing `uploadToR2()` function from `src/lib/storage/r2-client.ts`.

**R2 Key Pattern**: `reports/{userId}/{reportType}/{reportId}.{ext}`
- Example: `reports/user123/PNL/rpt456.csv`
- Example: `reports/user123/BALANCE_SHEET/rpt789.html`

Content types:
- CSV: `text/csv`
- HTML (PDF): `text/html`

The `fileUrl` stored in the `financialReport` row is the full R2 public URL returned by `uploadToR2()`.

1. **DO NOT** install Puppeteer, wkhtmltopdf, jsPDF, or any headless browser package. PDF export means "print-optimized HTML" in this context.
2. **DO NOT** create new database tables. The `financialReport` table already exists.
3. **DO NOT** create new database migrations. No schema changes needed.
4. **DO NOT** add `TAX_PREP` or `INVENTORY_AGING` report generation -- those are out of scope for D4.3. The validation schema restricts to PNL, BALANCE_SHEET, CASH_FLOW only.

## ACCEPTANCE CRITERIA

### Functional
1. A Finance PRO seller can generate a P&L report for any date range and see a detailed breakdown of revenue, COGS, fees (TF, payment processing, boost, insertion, local, auth, subscription), shipping, expenses by category, mileage deductions, and net earnings.
2. A Finance PRO seller can generate a balance sheet showing current assets (available for payout, pending/escrow, inventory value), liabilities (reserved funds), and equity.
3. A Finance PRO seller can generate a cash flow statement showing operating activities (sales, refunds, fees, expenses) and financing activities (payouts).
4. All three report types can be exported as CSV download.
5. All three report types can be exported as printable HTML (labeled as "PDF" in format column).
6. All three report types can be viewed inline (JSON format) without downloading.
7. Generated reports are persisted in the `financialReport` table with full snapshot data.
8. Previously generated reports appear in a list with download/view/delete actions.
9. Deleting a report also deletes the associated file from R2 (if fileUrl exists).
10. The statements page shows the `FinanceProGate` for FREE tier sellers.

### Authorization
11. Unauthenticated users are redirected to login.
12. FREE tier sellers see the `FinanceProGate` upgrade card, not the report generator.
13. Delegated staff with `finance.view` scope can view the list of generated reports and view/download them, but CANNOT generate new reports or delete existing ones.
14. Delegated staff without `finance.view` scope cannot access the statements page at all.
15. Users cannot access another user's reports (ownership enforced via `userId` condition in all queries).

### Data Integrity
16. All monetary values in report snapshots are stored as integer cents.
17. All monetary values displayed in CSV/HTML are formatted as dollars with two decimal places.
18. P&L net earnings = grossRevenue - COGS - platformFees - shipping - expenses - mileage.
19. Balance sheet equation: totalAssets = totalLiabilities + totalEquity.
20. Cash flow: netCashChange = netOperating + netFinancing.

### Vocabulary
21. No banned terms appear anywhere in UI text, CSV output, HTML output, or code comments.
22. Transaction Fee is labeled "Transaction Fee" (not "FVF", not "Commission").
23. Stripe fee is labeled "Payment processing fee".
24. Balance is labeled "Available for payout" (not "Twicely Balance").
25. Net result is labeled "Net earnings" (not "Net payout").

### Performance
26. Report generation queries use indexed columns and complete within reasonable time for sellers with up to 10,000 orders.
27. Report list query is paginated and does not load snapshot data (select only metadata columns).

function csvEscape(value: string | number): string {
  const str = String(value);
  if (str.includes(',') || str.includes('"') || str.includes('\n')) {
    return `"${str.replace(/"/g, '""')}"`;
  }
  return `"${str}"`;
}

function csvRow(...cells: Array<string | number>): string {
  return cells.map(csvEscape).join(',');
}

```typescript
import { headers } from 'next/headers';
import { redirect } from 'next/navigation';
import type { Metadata } from 'next';
import { auth } from '@/lib/auth';
import { getFinanceTier } from '@/lib/queries/finance-center';
import { getPlatformSetting } from '@/lib/queries/platform-settings';
import { FinanceProGate } from '@/components/finance/finance-pro-gate';
import { ReportGenerator } from '@/components/finance/report-generator';
import { ReportList } from '@/components/finance/report-list';

export const dynamic = 'force-dynamic';

export const metadata: Metadata = {
  title: 'Financial Statements | Twicely',
  robots: 'noindex',
};

export default async function StatementsPage() {
  const session = await auth.api.getSession({ headers: await headers() });

  if (!session?.user) {
    redirect('/auth/login?callbackUrl=/my/selling/finances/statements');
  }

  const [financeTier, annualCents, monthlyCents, retentionDaysFree, retentionYearsPro] =
    await Promise.all([
      getFinanceTier(session.user.id),
      getPlatformSetting<number>('finance.pricing.pro.annualCents', 999),
      getPlatformSetting<number>('finance.pricing.pro.monthlyCents', 1499),
      getPlatformSetting<number>('finance.reportRetentionDays.free', 30),
      getPlatformSetting<number>('finance.reportRetentionYears.pro', 2),
    ]);

  return (
    <div className="space-y-6">
      <div>
        <h1 className="text-2xl font-bold">Financial Statements</h1>
        <p className="text-muted-foreground text-sm">
          Generate P&amp;L reports, balance sheets, and cash flow statements
        </p>
      </div>

      {financeTier === 'PRO' ? (
        <>
          <ReportGenerator />
          <ReportList />
        </>
      ) : (
        <FinanceProGate
          annualMonthlyCents={annualCents}
          monthlyMonthlyCents={monthlyCents}
          retentionDaysFree={retentionDaysFree}
          retentionYearsPro={retentionYearsPro}
        />
      )}
    </div>
  );
}
```

## NOTES FOR THE INSTALLER

1. **"PDF" means HTML for printing.** The `format` column stores `'PDF'` but the actual file uploaded to R2 has a `.html` extension with `text/html` content type. This is intentional. Users download and print via browser. No headless browser needed.

2. **Balance sheet is simplified.** This is a cash-basis balance sheet for individual resellers, not a full accrual-basis statement. Inventory is valued at listing price (not cost basis). "Assets" means money + inventory. "Liabilities" means reserved/held funds. "Equity" is the difference. This is sufficient for the target audience.

3. **Beginning balance approximation.** Since we don't snapshot `sellerBalance` historically, the cash flow statement derives beginning balance as `endingBalance - netCashChange`. This is a known simplification. Accurate historical snapshots would require a cron job or event-sourced balance history, which is out of scope.

4. **No `finance.manage` delegation scope exists.** The Actors Canonical only defines `finance.view`. Report generation is therefore owner-only. This is correct -- staff should not generate financial reports on behalf of the seller without the seller's direct involvement.

5. **Report data is frozen at generation time.** The `snapshotJson` captures the state at generation. If the seller later adds/edits expenses, the saved report does NOT update. The seller must generate a new report to reflect changes. This is standard accounting behavior.
