## G3.7 Vacation Mode — Codebase Analysis (2026-03-14)

### What Already Exists (Substantial!)
- `sellerProfile` columns: vacationMode, vacationMessage, vacationStartAt, vacationEndAt, vacationModeType (5 columns)
- `storefront` columns: vacationMode, vacationMessage (mirror for query convenience)
- `VacationBanner` component at `src/components/storefront/vacation-banner.tsx` (simple, needs enhancement)
- `processVacationAutoEnd()` cron at `src/lib/commerce/vacation-cron.ts` (clears expired vacations)
- Cron registered in `src/lib/jobs/cron-jobs.ts` — midnight UTC, task='vacation'
- `checkListingAvailability()` handles SELLER_ON_VACATION + SELLER_ON_VACATION_ALLOW_SALES
- Offer blocking in `offer-create.ts` — checks `sellerProfile.vacationMode` before allowing new offers
- Cart blocking via `checkListingAvailability()` in `addToCart()`
- Storefront layout shows VacationBanner conditionally
- PublicStorefront shows inline vacation warning (amber box)
- `StorefrontOwnerData` already includes vacationMode, vacationMessage, vacationStartAt, vacationEndAt
- `StorefrontSeller` (public) already includes vacationMode, vacationMessage
- Seed data has `vacationMode: false` on all storefronts and seller profiles

### What Does NOT Exist
- No server action to activate/deactivate vacation (the core missing piece)
- No Zod validation schema for vacation input
- No `vacationAutoReplyMessage` column on sellerProfile (schema gap — owner decision needed)
- No messaging auto-reply integration (messaging-actions.ts has no vacation check)
- No pending-offer auto-decline on activation (only blocks new offers, not existing PENDING)
- No unfulfilled-order warning query
- No admin force-deactivate action
- No vacation platform settings seeded (4 keys needed)
- No vacation settings UI in store settings page
- No checkout buyer acknowledgment for ALLOW_SALES mode
- No Typesense search exclusion for PAUSE_SALES

### Key Architecture Facts
- `vacationModeType` is plain TEXT (not enum) — values: PAUSE_SALES | ALLOW_SALES | CUSTOM
- Storefront vacation fields are a MIRROR of sellerProfile — must sync both
- `declineAllPendingOffersForListing()` exists in offer-transitions.ts — can be reused for batch decline
- message table has `isAutoGenerated` boolean — perfect for auto-reply messages
- 4 platform settings from Feature Lock-in §30: vacation.maxPauseDays (30), vacation.maxAllowSalesDays (15), vacation.autoDeclinePendingOffers (true), vacation.requireBuyerAcknowledgment (true)
- No vacation settings exist in Platform Settings Canonical — only in Feature Lock-in

### Open Decisions
1. Add `vacationAutoReplyMessage` column? Schema doc doesn't have it, but Feature Lock-in implies separate message.
2. Include checkout ALLOW_SALES acknowledgment in G3.7 or defer?
3. Typesense search exclusion for PAUSE_SALES — G3.7 or separate?
4. Future-dated startAt — immediate activation or scheduled job?
